## 0. 서비스 정체성

- **서비스명**: 번개가입
- **운영사**: 주식회사 수파레인
- **플랫폼**: 카페24 공식 앱 (App Store 등록)
- **핵심 가치**: 카페24 쇼핑몰에 소셜 1클릭 가입을 제공. 운영자는 OAuth 개발자 앱 등록·심사 없이 즉시 사용 가능
- **고객지원 답변 톤**: 존댓말, 결론 먼저, 간결(5~10문단), 마지막은 "감사합니다. / 번개가입 드림 ⚡"

## 4. 회원 관리 주의사항

- 소셜 가입 고객은 카페24 회원 DB에 **이메일 ID 기준으로 등록** (이메일 없는 Telegram은 별도 규칙). 기존 이메일 회원이 있으면 자동 연동
- 소셜 회원에는 **`@s` 접미사**가 붙음 (예: `user@example.com@s`) — 카페24 기본 소셜 회원과 구분됨
- 소셜 회원 탈퇴 시 카페24 회원 정보는 함께 삭제됨 (개인정보처리방침 기준)
- **현재 소셜 가입 시 회원등급(일반/사업자)을 고객이 선택하는 UI는 미지원**. 모든 소셜 회원은 쇼핑몰의 "기본 회원등급"으로 자동 등록됨. 사업자 전환 필요 시 카페24 관리자 > 회원 > 회원등급 변경에서 수동 승격

## 5. 개인정보 수집 범위 (개인정보처리방침 기준)

**운영자(쇼핑몰 사장님)**: 이메일, 이름, 비밀번호(PBKDF2 100k 해시)
**고객(쇼핑몰 회원)**: OAuth provider ID, 이메일, 닉네임, 프로필 사진 URL
- Telegram은 이메일 없이 provider ID + 사용자명만 수집
**암호화**: PII는 AES-GCM, 비밀번호는 PBKDF2 100,000회, 검색용 해시는 SHA-256

## 6. 자주 묻는 운영 이슈 & 표준 답변 패턴

### "로그인 페이지에 소셜 버튼이 안 보여요"
→ 위젯 삽입 위치 확인 권장. "로그인 폼 아래" 또는 커스텀 셀렉터(`.login__button`, `.login__sns`)로 변경 시도

### "다른 SSO 앱에서 번개가입으로 전환하면 기존 회원은?"
→ 동일 이메일이면 기존 회원과 자동 연동. 별도 마이그레이션 작업 불필요

### "앱을 삭제하면 기존 소셜 회원은?"
→ 카페24 회원 DB의 회원 정보는 그대로 유지. 다만 소셜 버튼으로의 재로그인은 불가해지므로, 이메일 로그인·비밀번호 재설정 안내를 고객에게 발송 권장

### "사업자회원으로 소셜 가입 가능한가요?"
→ 현재 미지원 (모든 소셜 가입자는 기본 회원등급). 대안: (a) 소셜은 일반회원 전용, 사업자는 기존 가입 폼 유지 / (b) 소셜 가입 후 운영자가 수동 승격

### "구글/네이버 로그인 시 '번개가입으로 이동' 문구가 뜹니다"
→ 번개가입 앱이 자체 렌더링하는 화면에는 해당 문구 없음. 해당 문구는 (1) 구글/네이버 OAuth 동의 화면의 앱 이름 표시 또는 (2) 카페24 SSO 중간 안내 페이지 중 하나로 추정. 고객이 본 정확한 문구 전문과 표시 시점(버튼 클릭 직후 / 로그인 완료 후 등) 을 되묻기로 원인 특정 필요

### "카카오톡 채널 자동 연결"
→ **v2.5.0부터 Free 플랜에서도 사용 가능**. 가입 완료 후 쇼핑몰의 카카오톡 채널 추가를 유도. 설정 위치는 **기본 설정 페이지 최상단의 "카카오 채널" 카드** (`/dashboard/settings/general#kakaoChannelCard`). 카카오 채널 ID(밑줄로 시작하는 식별자)를 입력하면 동작, 비워두면 자동 비활성화.
- v2.5.1: 별도 사이드 메뉴 항목이 제거되고 기본 설정 페이지로 통합됨. 기존 URL `/dashboard/settings/kakao` 는 자동 리다이렉트.

### "가입/로그인 후 원래 페이지로 돌아오나요?" (v2.5.1)
→ 네, 자동 복귀. 예: 상품 페이지 → 로그인 → 가입 → 같은 상품 페이지로 자동 이동. URL 파라미터 + localStorage 폴백으로 구현.

### "프로바이더가 너무 많아요. 일부만 풀버튼, 나머지는 작은 아이콘으로?" (v2.5.0)
→ **아이콘 모드 프로바이더** 사용. 로그인 디자인 페이지 각 row 우측 `□ 아이콘` 체크박스를 켜면 해당 프로바이더는 풀버튼 영역 아래 44×44 원형 아이콘으로 표시. last-used override 유지(직전 로그인 프로바이더는 모드 무관 풀버튼 promote). icon-only preset 선택 시 모든 프로바이더 아이콘 처리되어 토글 비활성화.

### "위젯 안에 쿠폰팩이나 안내 문구를 넣고 싶어요" (v2.5.1, Plus)
→ Plus 전용. 로그인 디자인 > 추가 옵션 섹션에서:
- **쿠폰팩 카드**: 노출 위치(소셜 위/아래), 간격(0~60px), 크기(LG/MD/SM/XS), 디자인 4종(다크/브랜드/일러스트/미니멀), 반짝. 이탈 팝업 쿠폰팩과 디자인 동기화.
- **텍스트1**(상단 타이틀 아래 작은 글씨): 가입/로그인 편의성 강조. 기본 "아이디 비밀번호 입력없이 번개가입! 번개로그인!"
- **텍스트2**(소셜~쿠폰팩 사이 큰 볼드): 쿠폰팩 증정 강조 권장. 기본 "회원가입 즉시 사용가능한 쿠폰팩 증정"
- AI 추천 문구도 이 두 영역에 자동 적용 가능.

## 6-7. 문의 게시판 이미지 첨부 기능 (Phase 2)

- 문의 작성 시 이미지를 최대 5개 첨부할 수 있습니다 (파일당 5MB 이하, PNG/JPEG/WebP/GIF).
- 첨부 방법: 문의 작성 폼 하단의 이미지 업로드 영역에 클릭하거나 드래그&드롭으로 추가.
- 운영자가 문의 내용 이해에 도움이 되는 화면 캡처·오류 메시지 이미지를 첨부하면 더 빠른 답변이 가능합니다.

## 7. 답변 시 주의 (LLM 금지 사항)

**절대 답변에 포함 금지**:
- 내부 테이블명·컬럼명 (`shops`, `inquiries`, `user_providers` 등)
- API 엔드포인트 경로 (`/api/supadmin/...`, `/api/dashboard/...`)
- D1 database_id, KV namespace, Cloudflare account_id
- OAuth client_id / client_secret / redirect_uri 구체 값
- 내부 환경변수명 (`JWT_SECRET`, `ENCRYPTION_KEY` 등)
- 본 KB에 **명시되지 않은 추측성 내용** → 반드시 "해당 부분은 확인 후 안내드리겠습니다"로 대응

**우회 표현 가이드**:
- "토큰 교환" → "로그인 처리"
- "웹훅 수신" → "자동 연동"
- "JWT/세션 만료" → "로그인 상태 유지 시간"
- "KV 캐시" → 언급 자체 금지

## 1. 시스템 아키텍처

### 1.1 기술 스택

| 구성요소 | 기술 | 비고 |
|---------|------|------|
| **런타임** | Cloudflare Workers | 서버리스, 글로벌 엣지 |
| **언어** | TypeScript | Workers 공식 지원 |
| **웹 프레임워크** | Hono | 경량 Workers 전용 프레임워크 |
| **DB (영구)** | Cloudflare D1 | SQLite 기반 |
| **DB (임시/캐시)** | Cloudflare KV | 토큰/세션/코드 (TTL 활용) |
| **카페24 연동** | `@supasignup/cafe24-client` | TypeScript 클라이언트 (Workers 호환) |
| **프론트 (위젯)** | Vanilla JS (IIFE) | ScriptTag로 삽입, Shadow DOM, localStorage |
| **대시보드** | Workers SSR (Hono JSX) | Server-Side Rendering |
| **AI 기능** | Cloudflare Workers AI (Kimi K2.5) | Plus 전용 |
| **모노레포** | pnpm workspaces | `workers/api`, `packages/cafe24-client`, `packages/bg-core` |

## 2. API 명세

### 2.1 OAuth 엔드포인트

**베이스 경로**: `/oauth`  
**인증**: 없음 (public, 카페24 SSO 스펙)

| 상태 | 메서드 | 경로 | 설명 |
|------|--------|------|------|
| ✅ | GET | `/oauth/authorize` | OAuth 플로우 시작. 소셜 Provider로 리다이렉트 |
| ✅ | GET | `/oauth/callback/:provider` | 소셜 콜백 처리. 사용자 저장 + 카페24 코드 반환 |
| ✅ | POST | `/oauth/token` | 카페24가 호출. 코드 → 액세스 토큰 교환 |
| ✅ | GET | `/oauth/userinfo` | 카페24가 호출. 토큰 → 사용자 정보 반환 |

**GET /oauth/authorize 파라미터:**
```
client_id     (required) — shops.client_id
redirect_uri  (required) — 화이트리스트 검증
state         (required) — 카페24 CSRF 토큰
provider      (optional) — KV hint에서 자동 감지
```

**GET /oauth/callback/:provider 처리 흐름:**
1. 소셜 OAuth 코드 수신
2. 소셜 토큰 교환 → 사용자 정보 획득
3. D1에 user upsert (PII AES-GCM 암호화)
4. shop_users 생성 (신규 가입 판별)
5. login_stats 기록
6. sso_type 자동 감지 (redirect_uri 패턴 분석)
7. 카페24 SSO 콜백으로 Authorization Code 반환 (KV 저장, 5분 TTL)

**지원 Provider 목록:** `google`, `kakao`, `naver`, `apple`, `discord`, `facebook`, `x`, `line`, `telegram`

### 2.6 과금 API

**베이스 경로**: `/api/dashboard/billing`  
**인증**: 필수

| 상태 | 메서드 | 경로 | 설명 |
|------|--------|------|------|
| ✅ | POST | `/api/dashboard/billing/subscribe` | 구독 주문 생성 (카페24 인앱결제) |
| ✅ | GET | `/api/dashboard/billing/return` | 결제 완료 후 리다이렉트 페이지 |
| ✅ | POST | `/api/dashboard/billing/cancel` | pending 구독 취소 |
| ✅ | GET | `/api/dashboard/billing/status/:id` | 구독 상태 조회 |

**결제 플로우:**
```
POST /billing/subscribe
  → Cafe24 Appstore Order 생성 (createAppstoreOrder)
  → confirmation_url 반환
  → 사용자가 카페24 결제 팝업에서 결제
  → 카페24 웹훅 POST /api/cafe24/webhook (event_no=90157)
  → subscriptions 활성화 + shops.plan 업데이트
```

**레이스 컨디션 대응**: 웹훅이 `payment_id` 저장보다 먼저 도착할 경우 KV에 임시 저장 (`webhook:payment:{order_id}`, 5분 TTL). `/subscribe` 응답 전에 KV 확인하여 즉시 처리.

## 3. 데이터 모델

### 3.1 현재 D1 스키마 (구현 완료)

**owners** — 운영자 계정
```sql
owner_id     TEXT PRIMARY KEY          -- nanoid
email        TEXT NOT NULL UNIQUE
name         TEXT
password_hash TEXT NOT NULL            -- bcrypt
role         TEXT NOT NULL DEFAULT 'user'  -- 'user' | 'admin'
deleted_at   TEXT                      -- soft delete
created_at   TEXT NOT NULL DEFAULT (datetime('now'))
updated_at   TEXT NOT NULL DEFAULT (datetime('now'))
```

**shops** — 등록된 쇼핑몰
```sql
shop_id                TEXT PRIMARY KEY
mall_id                TEXT NOT NULL
platform               TEXT NOT NULL    -- 'cafe24' | 'imweb' | 'godomall' | 'shopby'
shop_name              TEXT
shop_url               TEXT
owner_id               TEXT NOT NULL REFERENCES owners(owner_id)
client_id              TEXT NOT NULL UNIQUE    -- 번개가입 발급 key
client_secret          TEXT NOT NULL           -- 번개가입 발급 secret
enabled_providers      TEXT NOT NULL DEFAULT '["google","kakao","naver","apple"]'  -- JSON array
platform_access_token  TEXT                    -- AES-GCM 암호화
platform_refresh_token TEXT                    -- AES-GCM 암호화
allowed_redirect_uris  TEXT                    -- JSON array (화이트리스트)
plan                   TEXT NOT NULL DEFAULT 'free'  -- 'free' | 'monthly' | 'yearly'
sso_configured         INTEGER NOT NULL DEFAULT 0
widget_style           TEXT                    -- JSON: WidgetStyle
sso_type               TEXT NOT NULL DEFAULT 'sso'  -- 'sso' | 'sso1' | 'sso2' (자동 감지)
deleted_at             TEXT
created_at             TEXT NOT NULL DEFAULT (datetime('now'))
updated_at             TEXT NOT NULL DEFAULT (datetime('now'))
UNIQUE(mall_id, platform)
```

**users** — 소셜 인증 사용자 (PII 암호화)
```sql
user_id       TEXT PRIMARY KEY
provider      TEXT NOT NULL
provider_uid  TEXT NOT NULL
email         TEXT       -- AES-GCM 암호화
email_hash    TEXT       -- SHA-256 해시 (검색용, 평문 노출 없음)
name          TEXT       -- AES-GCM 암호화
profile_image TEXT       -- URL (평문)
raw_data      TEXT       -- AES-GCM 암호화 JSON (전체 소셜 응답)
phone         TEXT       -- AES-GCM 암호화
birthday      TEXT       -- AES-GCM 암호화
gender        TEXT       -- 평문 (민감도 낮음)
created_at    TEXT NOT NULL DEFAULT (datetime('now'))
updated_at    TEXT NOT NULL DEFAULT (datetime('now'))
UNIQUE(provider, provider_uid)
```

**shop_users** — 쇼핑몰-사용자 매핑

## 4. ScriptTag 위젯 아키텍처

### 4.1 현재 구현 — buttons.js (✅ 구현 완료)

**서빙 경로**: `GET /widget/buttons.js`  
**클라이언트 식별**: `?shop={client_id}` 쿼리 파라미터  
**캐싱**: ETag 기반 (코드 변경 시 자동 무효화), `Cache-Control: no-cache`

**주요 구조:**
```
IIFE (즉시 실행 함수)
└── BGWidget 클래스 (prototype 기반)
    ├── init()            — CSS 주입, client_id 추출, localStorage 읽기
    ├── detectPageType()  — URL 패턴으로 login/myshop/other 감지
    ├── loadConfig()      — /api/widget/config API 호출
    ├── render()          — Shadow DOM 없이 DOM 직접 조작
    ├── renderLoginPage() — 로그인/가입 페이지 버튼 렌더링
    ├── renderMyshop()    — 마이페이지 소셜 연동 위젯 렌더링
    ├── handleLogin()     — KV hint 저장 → 카페24 SSO 트리거
    ├── handleDirectOAuth() — SSO 없이 직접 OAuth (Discord 등)
    └── saveLastProvider() — URL 파라미터에서 provider 추출 → localStorage
```

**스마트 버튼 로직:**
- 가입/로그인 성공 후 `bg_last_provider`를 `localStorage`에 저장
- 재방문 시 해당 버튼에 `bg-btn-highlight` 클래스 추가 + "이전에 사용" 배지
- 멀티 쇼핑몰 지원: `bg.suparain.kr` 도메인에서 동일 localStorage 공유

**SSO 플로우 (Google/Kakao/Naver/Apple):**

**직접 OAuth (Discord/Facebook/LINE/X/Telegram):**

## 6. 카페24 연동 상세

### 6.1 카페24 앱 OAuth 플로우 (✅ 구현 완료)

```
운영자 → 카페24 앱스토어 설치
  → GET /api/cafe24/install?mall_id=xxx&hmac=xxx&timestamp=xxx
      HMAC-SHA256 검증 (5분 유효)
      CSRF state 생성 → KV 저장
  → 카페24 OAuth URL로 리다이렉트 (15개 scope)
  → GET /api/cafe24/callback?code=xxx&state=mallId:csrfToken
      CSRF state KV 검증 후 즉시 삭제 (재사용 방지)
      Cafe24Client.exchangeCode() — 토큰 교환
      Cafe24Client.getStoreInfo() — 쇼핑몰 정보 조회
      createShop() or updateShop() — D1 저장
          platform_access_token/refresh_token — AES-GCM 암호화 저장
          allowed_redirect_uris — 자동 생성 (sso, sso1, sso2 슬롯 포함)
      createScriptTag() — buttons.js 설치
  → 대시보드 setup 페이지로 리다이렉트 (JWT 쿠키 자동 발급)
```

**재설치 처리**: `deleted_at IS NOT NULL` 쇼핑몰 감지 → soft delete 복원 + 토큰 갱신

### 6.2 SSO 플로우 (✅ 구현 완료)

카페24 SSO는 최대 5개 슬롯 (`sso`, `sso1`, `sso2`, `sso3`, `sso4`).  
현재 Google/Kakao/Naver/Apple 4개 SSO 등록.

```
[카페24 로그인 페이지에서 SSO 버튼 클릭]
  → 카페24가 /oauth/authorize?client_id=xxx&redirect_uri=...&state=cafe24state&provider=google 호출
  → provider가 없으면 KV hint 조회 (widget이 미리 저장)
  → 소셜 OAuth URL로 즉시 리다이렉트 (중간 선택 페이지 없음)
  → 소셜 OAuth 완료 → GET /oauth/callback/google
  → D1 upsert + login_stats 기록
  → 카페24 SSO 콜백 URL로 Authorization Code 반환

[카페24가 후속 API 호출]
  → POST /oauth/token (code → access_token, KV 저장 2시간 TTL)
  → GET /oauth/userinfo (access_token → 사용자 정보 반환)
  → 카페24가 회원 자동 생성 + 로그인 세션 생성
```

**sso_type 자동 감지**: `redirect_uri` 패턴에서 `/Oauth2ClientCallback/(sso[12]?)/` 추출 → `shops.sso_type` 업데이트 (비동기, `waitUntil`으로 응답 차단 없음)

## 8. 보안

### 8.1 PII 암호화 (✅ 구현 완료)

**알고리즘**: AES-256-GCM (Web Crypto API, Workers 네이티브)  
**암호화 대상**: `users.email`, `users.name`, `users.raw_data`, `users.phone`, `users.birthday`  
**검색용 해시**: `users.email_hash` (SHA-256, 평문 이메일 노출 없이 조회 가능)  
**키 관리**: `ENCRYPTION_KEY` Workers Secret (환경별 반드시 다른 키)

```typescript
// packages/bg-core/src/crypto.ts
encrypt(plaintext: string, key: string): Promise<string>  // AES-GCM
decrypt(ciphertext: string, key: string): Promise<string>
```

---

### 8.2 JWT 인증 (✅ 구현 완료)

**라이브러리**: Workers 네이티브 Web Crypto (외부 의존 없음)  
**토큰 저장**: `bg_token` 쿠키 (`HttpOnly; Secure; SameSite=Strict`)  
**유효 기간**: 24시간 (`Max-Age=86400`)  
**미들웨어**: `authMiddleware` (Bearer 헤더 또는 쿠키 자동 감지)

---

### 8.3 CORS 동적 처리 (✅ 구현 완료)

```typescript
// 위젯 API: 전체 오픈 (쇼핑몰에서 호출)
app.use('/api/widget/*', cors({ origin: '*', allowMethods: ['GET'] }));

// 대시보드 API: BASE_URL 동일 출처만
app.use('/api/dashboard/*', cors({
  origin: (origin) => origin === BASE_URL ? origin : '',
  allowMethods: ['GET', 'POST', 'PUT', 'DELETE'],
}));
```

---

### 8.4 redirect_uri 화이트리스트 (✅ 구현 완료)

앱 설치 시 `allowed_redirect_uris` 자동 생성:
```
https://{mall_id}.cafe24api.com/api/v2/oauth/callback
https://{mall_id}.cafe24.com/Api/Member/Oauth2ClientCallback/sso/
https://{mall_id}.cafe24.com/Api/Member/Oauth2ClientCallback/sso1/
https://{mall_id}.cafe24.com/Api/Member/Oauth2ClientCallback/sso2/
https://{base_url}/link/complete
https://{custom_domain}/Api/Member/Oauth2ClientCallback/sso/  (커스텀 도메인 있는 경우)
```

`/oauth/authorize` 호출 시 `redirect_uri` 화이트리스트 검증 (없으면 400 에러).

---

### 8.5 OAuth PKCE (✅ 구현 완료)

소셜 OAuth 플로우에 PKCE 적용:
- `code_verifier`: 랜덤 64바이트 Base64URL
- `code_challenge`: SHA-256 해시 Base64URL
- KV에 세션 저장 (10분 TTL), 콜백에서 검증 후 삭제

---

### 8.6 SSO type 자동 감지 (✅ 구현 완료)

`/oauth/authorize` 요청의 `redirect_uri`에서 sso 슬롯 패턴 감지:
```
/Oauth2ClientCallback/sso/   → sso_type = 'sso'
/Oauth2ClientCallback/sso1/  → sso_type = 'sso1'
/Oauth2ClientCallback/sso2/  → sso_type = 'sso2'
```
DB + KV 캐시 비동기 업데이트 (`waitUntil`, 응답 차단 없음).

---

### 8.7 기타 보안 설계

| 위협 | 대응 방법 |
|------|---------|
| CSRF (앱 설치) | 랜덤 state 토큰 + KV 검증 후 즉시 삭제 |
| 웹훅 위조 | HMAC-SHA256 서명 검증 또는 x-api-key 검증 |
| 타이밍 공격 | `timingSafeEqual()` 사용 (bg-core 유틸) |
| Rate Limit | `rateLimitMiddleware` (auth 엔드포인트) |
| 쿠폰 중복 발급 | `shop_users` UNIQUE(shop_id, user_id) 제약으로 신규 가입 판별 |
| 전역 에러 핸들러 | 스택 트레이스 응답 노출 방지 (`app.onError`) |
| Provider hint 주입 | Origin/Referer 헤더로 요청 출처 검증 |

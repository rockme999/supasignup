카페24 파트너센터 **프로모션 메뉴 > 스토리 / 사용팁**에 게시할 콘텐츠와 관련 이미지 자산을 이 폴더에서 관리한다.

업로드 화면(참고):
- 스토리: `docs/cafe24-story-input.html`
- 사용팁: `docs/cafe24-usetip-input.html`

```

├── tips/                             # 사용팁 (TIP) 콘텐츠
│   ├── tip-01-install-quickstart.md
│   ├── tip-01-install-quickstart.html

```

## 카페24 입력 폼 필드 요약

### 공통 (스토리 / 사용팁)

| 필드 | 제약 | 비고 |
|---|---|---|
| 스토어 선택 | 스토어/디자인센터, KR/JP/GL | **최초 저장 후 변경 불가** — 한국 스토어로 |
| 제목 (`contents_name`) | **최대 80 Bytes** | UTF-8 기준 한글 ≈ 26자 |
| 요약 설명 (`contents_desc`) | **최대 120 Bytes** | UTF-8 기준 한글 ≈ 40자 |
| 대표 이미지 | 444×320 px, 500KB 이하, png/jpg | 또는 YouTube 동영상 URL 선택 가능 |
| 노출 토글 | 심사 완료 후에만 활성화 | 기본 "노출안함" |
| 관련 상품 | 선택 사항 | 번개가입 본 앱 선택 |
| 상세 내용 HTML | 에디터 or 직접 HTML | 본문 **이미지 가로 720px 이하**, `alt` 필수 |

### 본문 HTML 작성 규칙 (Froala 에디터 호환)

1. `<style>` / `<script>` 태그는 Froala가 필터링할 수 있음 → **인라인 스타일(`style="..."`) 사용**.
2. 이미지 태그는 반드시 `alt="설명"` 포함 (웹접근성 의무).
3. 본문 이미지 가로 최대 720px. 표(table)도 720px 초과 금지.
4. 한 번에 업로드 가능한 이미지 10MB 이하.
5. 자체 호스팅 이미지 URL 대신, **카페24 에디터에 직접 업로드**해 카페24 CDN을 사용한다(외부 링크는 심사에서 걸릴 수 있음).

### 심사 기준 (요약)

**스토리**:
- 매출 증가·운영 효율 등 **쇼핑몰 운영자에게 유익한 내용**
- 시각적 요소(이미지·동영상)와 이야기 전개
- 기존 게시물과 유사·중복되지 않아야 함
- 타 서비스·타사 가격 비교 시 근거 명확해야 함

**사용팁**:
- 상품상세정보와 같은 **단순 기능 나열 금지**
- 설치·사용 방법, CS 해결, 고객 만족 향상에 도움이 되는 구체적 내용
- 단계별 절차, 스크린샷으로 실제 사용법을 보여줄 것

## 작업 흐름

1. **콘텐츠 기획** — `stories/*.md` 또는 `tips/*.md`에 메타데이터와 스토리보드 작성.
2. **본문 HTML 작성** — 같은 이름의 `.html` 파일에 Froala에 그대로 붙여넣을 수 있는 self-contained HTML 작성. 이미지는 일단 플레이스홀더 경로(`images/screenshots/xxx.png`)로 기입.
3. **이미지 캡쳐** — 사용자가 관리자 화면을 캡쳐해 `images/screenshots/`에 저장. 대표 이미지는 `images/main/`에 444×320 사이즈로.
4. **카페24 업로드** — 파트너센터 스토리 등록 화면에서:
   - 제목·요약 복사 붙여넣기
   - 대표이미지 파일 업로드 (`images/main/story-01-main.png` 등)
   - 본문 HTML을 Froala 에디터의 "코드 보기" 모드로 붙여넣기
   - 본문 이미지는 에디터에서 직접 업로드해 카페24 CDN 경로로 교체
5. **심사 요청** — 저장 후 "심사요청" 클릭. 반려 시 이 폴더의 소스를 수정해 재제출.

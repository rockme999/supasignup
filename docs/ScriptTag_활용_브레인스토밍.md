## 0. 전제 — 모바일 우선 설계

### 왜 모바일 우선인가

| 지표 | 수치 |
|------|------|
| 한국 이커머스 모바일 구매 비율 (2025) | **73.2%** |
| 2025년 전망 | **82%** |
| 모바일 쇼핑 거래액 | **182조원+** |

### 한국 모바일 이커머스의 특수 환경

| 환경 | ScriptTag 영향 |
|------|---------------|
| **카카오톡/네이버 인앱 브라우저** | localStorage 세션 간 미유지 가능, Share API 미지원 |
| **삼성 인터넷 (40대+)** | **광고 차단기 내장** — 클래스명에 ad/popup/banner 사용 시 필터링 위험 |
| **iOS Safari** | ITP 7일 쿠키 제한, Private 모드 localStorage 예외 |
| **Android Chrome** | 가장 안정적 — 기준 구현 대상 |

### ScriptTag 기술 제약

- ScriptTag는 **PC + 모바일 스킨 모두에 삽입됨**
- `display_location`으로 모바일/PC 구분 불가 → JS 런타임에서 분기
- 카페24 앱당 ScriptTag 수 제한 → **단일 loader.js로 모든 기능 관리**

### ScriptTag 필수 기술 스택

```
[반드시 사용]
- Shadow DOM (closed)     → CSS/DOM 격리
- IntersectionObserver    → 뷰포트 감지, 노출 추적
- MutationObserver        → SPA 페이지 전환 감지
- Beacon API              → 이탈 시 분석 데이터 전송
- visibilitychange        → 탭/앱 전환 감지
- matchMedia              → 모바일/PC/다크모드 분기
- visualViewport API      → 키보드/주소창 변화 대응
- localStorage + Cookie   → 이중 저장 (인앱 브라우저 대응)

[사용 금지]
- viewport meta 수정      → 호스트 페이지 레이아웃 파괴
- history.pushState 남용   → 쇼핑몰 라우팅 충돌
- Web Notification 요청    → 사용자 신뢰도 저하
- innerHTML               → XSS 위험
- "ad/banner/popup" 클래스명 → 광고 차단기 필터링
- CSS !important 남발     → 호스트 스타일 파괴
- 전역 변수 오염           → 네임스페이스 격리 필수
```

### 1.3 핵심 발견

1. **ScriptTag로 기존 DOM 자유 조작 가능** — 텍스트, 스타일, 애니메이션, 새 요소 삽입 모두 OK
2. **flex 컨테이너 주의** — `.action_button`이 flex라서 내부 삽입 시 옆으로 붙음 → **부모 레벨에 삽입** 또는 **버튼 내부 absolute**로 해결
3. **모바일 플로팅 버튼 별도 처리 필요** — `#orderFixArea`는 PC 본문 버튼과 별개. 모바일에서는 이 플로팅 버튼이 항상 보이므로 여기를 타겟해야 함
4. **페이지 구분 필수** — 장바구니의 "전체상품주문"과 상품상세의 "BUY IT NOW"이 같은 선택자 (`#orderFixArea a.btnSubmit`). URL/DOM 기반 페이지 판별 필요
5. **오버레이 뱃지 패턴 유효** — 작은 뱃지(알약형)가 버튼 위에 겹치면서 점멸 → 시각적으로 효과적이고 원래 버튼 기능 방해 안 함

### 1.4 카페24 상품 상세 DOM 맵

```
.headingArea h1                          ← 상품명
#span_product_price_custom               ← 소비자가 (취소선)
#span_product_price_text                 ← 판매가
.delv_price_B                            ← 배송비
#product_option_id1                      ← 옵션 선택 (select)
.action_button                           ← 버튼 컨테이너 (flex) — PC 본문
  a.btnSubmit > span#actionBuy           ← BUY IT NOW
  button#actionCart                       ← CART
  button#actionWish                       ← WISH LIST
#orderFixArea                            ← 모바일 하단 플로팅 컨테이너
  a.btnSubmit > span#actionBuy           ← BUY IT NOW (플로팅)
  button.actionCart                       ← CART (플로팅)
```

### 5.4 AI 활동 보고 시스템 — "AI 직원"

> 핵심 개념: AI를 "설치하고 잊는 도구"가 아니라, **매일 보고하는 마케팅 직원**으로 포지셔닝

#### 보고 체계

| 주기 | 보고 내용 | 대상 | 가치 |
|------|----------|------|------|
| **일일 보고** | AI가 수행한 홍보 활동 일지 | 관리자(쇼핑몰 운영자) | "오늘 뭘 했는지" 투명성 |
| **주간 보고** | 주간 성과 요약 (가입 수, 전환율, 클릭률) | 관리자 | "실제로 효과가 있는지" 증명 |
| **월간 보고** | 인사이트 + 경쟁업체 분석 + 개선 제안 | 관리자/경영진 | "다음에 뭘 해야 하는지" 전략 |

#### 일일 보고 예시

```
📋 번개가입 AI 일일 활동 보고 — 2026-04-15 (화)

[오늘의 활동]
• 상품상세 페이지: "3,000원 할인" 뱃지 노출 1,247회, 클릭 89회 (7.1%)
• 장바구니 페이지: 체류 10초+ 고객 42명에게 추가 쿠폰 제공 → 18명 주문 완료
• 미니배너: AI 생성 카피 "봄맞이 가입 혜택" 노출 → 기존 카피 대비 +23% 클릭률
• 소셜 가입: 카카오 31명, 네이버 12명, 구글 8명, 기타 4명

[오늘의 특이사항]
• 카카오 인앱 브라우저 유입 비율 41% (전주 대비 +5%)
• 모바일 전환율이 PC 대비 1.8배 높음
```

#### 주간 보고 예시

```
📊 번개가입 AI 주간 성과 보고 — 2026-04-09 ~ 04-15

[핵심 지표]
• 신규 가입: 385명 (전주 대비 +12%)
• 가입 전환율: 4.2% → 4.8% (+0.6%p)
• 가장 효과적인 효과: 가격 카운트다운 (전환율 6.1%)
• 가장 효과적인 프로바이더: 카카오 (전체의 57%)

[AI 최적화 결과]
• A/B 테스트: 배너 카피 B안 ("회원만 할인") 승리 → 자동 전환 완료
• 버튼 순서: 카카오→네이버→구글 순서가 최적 (자동 적용 중)
```

#### 월간 보고 예시

```
📈 번개가입 AI 월간 인사이트 — 2026년 4월

[성과 요약]
• 누적 가입: 1,580명 / 전환율: 4.8% / 월 매출 기여: 추정 ₩4,740,000

[인사이트]
• 화~목 오후 2~6시가 가입 전환율 최고 (7.2%) → 이 시간대 배너 강화 권장
• 모바일 사용자의 장바구니 체류 시간: 평균 23초 → 10초 에스컬레이션이 최적
• 카카오 가입자의 구매 전환율(32%)이 네이버(18%)보다 1.8배 높음

[경쟁 분석]
• 1초가입: 이번 달 앱스토어 리뷰 3건 추가 (부정 1건 — UI 오류 관련)
• 알파푸시: 새 기능 "AI 상품 추천" 출시 — 우리와 직접 경쟁 아님
• 시장 동향: 카페24 앱스토어 "소셜 로그인" 카테고리 검색량 +15%

[다음 달 제안]
• Discord 프로모션 강화 (MZ 유입 채널로 성장 중)
• 무료배송 게이지 활성화 권장 (장바구니 이탈 32% → 예상 20%로 감소)
```

#### 이 시스템의 전략적 의의

**1. 신뢰 구축 → 해지 방어**
- 운영자가 "이 앱이 실제로 일하고 있다"고 매일 느낌
- 경쟁사는 설치 후 아무런 피드백 없음 → "효과가 있나?" 의심 → 해지
- 번개가입은 매일 보고 → "이렇게 열심히 하는데 끊을 수 없지"

**2. 데이터 Lock-in 심화**
- 일일/주간/월간 보고가 쌓일수록 → "이 데이터를 버릴 수 없다"
- 3개월치 보고서 = 쇼핑몰의 고객 행동 패턴 자산
- 경쟁사로 이전하면 이 모든 인사이트가 소멸

**3. 경쟁사 불가 차별점**
- "AI가 일일보고하는 마케팅 도구" → 카페24 앱스토어에 전무
- 앱스토어 설명: "매일 보고하는 AI 마케터를 월 0원에 고용하세요"
- 이것 자체가 마케팅 메시지

**4. 상위 플랜 전환 근거**
- 무료 플랜: 기본 일일 요약만
- 유료 플랜: 상세 분석 + 경쟁 분석 + AI 제안
- 보고서 품질 차이 = 자연스러운 업그레이드 동기

#### Phase별 구현

| Phase | 보고 수준 | 구현 |
|-------|----------|------|
| **MVP (Phase 1)** | 기본 일일 요약 (가입 수, 클릭률) | 대시보드 내 간단 리포트 |
| **V2 (Phase 2)** | 주간 성과 보고 + AI 최적화 결과 | 이메일/카카오 알림톡 발송 |
| **V3 (Phase 3)** | 월간 인사이트 + 경쟁 분석 + AI 제안 | 자동 생성 리포트 (AI) |

## 8. 아키텍처 — loader.js 모듈 구조

### 8.1 전체 흐름

```
카페24 쇼핑몰 페이지 로드
  ↓
<script src="https://bg.suparain.kr/widget/loader.js?shop=SHOP_ID"></script>
  ↓
loader.js:
  1. 환경 감지 (모바일/PC, 인앱 브라우저, 다크모드)
  2. 페이지 판별 (상품상세 / 장바구니 / 결제 / 주문완료 / 메인 / 로그인)
  3. 로그인 상태 확인
  4. API 호출: GET /api/widget/config?shop=SHOP_ID
  5. 방문 횟수 확인 (localStorage)
  6. 활성화된 기능 모듈만 동적 로드
  7. 체류 시간 타이머 시작
  8. 각 모듈이 Shadow DOM 내에서 렌더링
```

### 8.2 모듈 구조

```
widget/
  loader.js              ← 진입점 (환경/페이지 감지 + 설정 로드 + 모듈 분기)
  modules/
    social-buttons.js     ← 소셜 로그인 버튼 (로그인 페이지)
    product-effects.js    ← 상품상세 효과 (A1~C3 전체)
    cart-effects.js       ← 장바구니 효과 (CA~CD 전체)
    checkout-effects.js   ← 주문서/주문완료 효과 (OA, PA)
    mini-banner.js        ← 슬라이드업/플로팅/탑바 배너 (메인/카테고리)
    coupon-toast.js       ← 가입 축하 + 쿠폰 안내 토스트
    escalation.js         ← 체류 시간 에스컬레이션 엔진
    tracker.js            ← 클릭/노출/전환 추적
  shared/
    env-detect.js         ← 모바일/PC, 인앱, 다크모드 감지
    page-detect.js        ← 페이지 유형 판별
    storage.js            ← localStorage + Cookie 이중 저장
    shadow-dom.js         ← Shadow DOM 컨테이너 관리
    safe-area.js          ← 노치/홈바/키보드 대응
```

### 8.3 페이지 판별 로직

```javascript
function detectPage() {
  var path = window.location.pathname;
  if (path.includes('/product/'))        return 'product';
  if (path.includes('/order/basket'))    return 'cart';
  if (path.includes('/order/orderform')) return 'checkout';
  if (path.includes('/order/order_result')) return 'complete';
  if (path.includes('/member/login'))    return 'login';
  if (path.includes('/member/join'))     return 'join';
  if (path === '/' || path === '')       return 'main';
  return 'other';
}
```

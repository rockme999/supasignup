## 1. 현재 구현 상태 요약

### ✅ 구현 완료 (건드리지 않을 것)

#### 인프라 & 기반

- 모노레포 구조 (`workers/api`, `packages/cafe24-client`, `packages/bg-core`)
- Cloudflare Workers + D1 + KV 환경 (스테이징/프로덕션 분리)
- `bg.suparain.kr` / `bg-dev.suparain.kr` 도메인 배포
- `wrangler.toml` 환경 분리 (`--env dev` / 기본)

#### D1 스키마 (현재 적용 완료)

- `owners`, `shops`, `users`, `shop_users`, `subscriptions`, `login_stats`, `user_providers`, `audit_logs`
- `shops.plan` CHECK: `('free', 'plus')` — v2 반영 완료
- `subscriptions.billing_cycle`: `('monthly', 'yearly')` — v2 반영 완료

#### OAuth 엔드포인트

- `GET /oauth/authorize` — 소셜 OAuth 시작, PKCE, KV 세션
- `GET /oauth/callback/:provider` — 소셜 콜백 처리 (9개 프로바이더)
- `POST /oauth/token` — 카페24 코드 → 토큰 교환 (일회용)
- `GET /oauth/userinfo` — 토큰 → 사용자 정보 반환

#### 위젯 (ScriptTag)

- `GET /widget/buttons.js` — 소셜 로그인 버튼 렌더링 (Shadow DOM 없음, IIFE)
- `GET /api/widget/config` — 위젯 설정 반환 (KV 캐시 5분)
- `GET /api/widget/hint` — 프로바이더 힌트 KV 저장 (cross-domain)
- 스마트 버튼 (이전 로그인 방법 기억 + 강조 표시)
- 소셜 9종 완전 지원 (SSO 4종 + 직접 OAuth 5종)

#### 카페24 연동

- `GET /api/cafe24/install` — HMAC 검증 + OAuth 리다이렉트
- `GET /api/cafe24/callback` — 토큰 교환 + 쇼핑몰 등록 + ScriptTag 등록
- `POST /api/cafe24/webhook` — 앱 삭제/만료/결제 웹훅 처리

#### 대시보드 API

- 인증: `POST /api/dashboard/auth/register`, `POST /api/dashboard/auth/login`
- 쇼핑몰 CRUD: `GET/POST/PUT/DELETE /api/dashboard/shops`
- 프로바이더 관리: `PUT /api/dashboard/shops/:id/providers`
- 위젯 스타일: `PUT /api/dashboard/shops/:id/widget-style`
- SSO 설정 정보: `GET /api/dashboard/shops/:id/setup`

#### 대시보드 API — 통계

- `GET /api/dashboard/stats` — 집계 통계 (today, 7d, 30d, month)
- `GET /api/dashboard/stats/daily` — 일별 추이
- `GET /api/dashboard/stats/export` — CSV 다운로드
- `GET /api/dashboard/stats/:shop_id` — 쇼핑몰별 통계
- `GET /api/dashboard/billing/status` — 플랜 상태/사용량

#### 과금 API

- `POST /api/dashboard/billing/subscribe` — 카페24 인앱결제 주문 생성
- `GET /api/dashboard/billing/return` — 결제 완료 처리
- `POST /api/dashboard/billing/cancel` — pending 구독 취소
- `GET /api/dashboard/billing/status/:id` — 구독 상태 조회
- 레이스 컨디션 대응 (KV 임시 저장)

#### 대시보드 앱 (SSR 페이지)

- `/` 랜딩 페이지
- `/privacy`, `/terms`
- `/dashboard/login`, `/dashboard/register`
- `/dashboard/shops`, `/dashboard/shops/new`
- `/dashboard/shops/:id/setup`, `/dashboard/shops/:id/providers`
- `/dashboard/billing`, `/dashboard/settings`
- `/dashboard/admin/*` (슈퍼어드민)

#### 3.2.1 백엔드 — 쿠폰 설정 API

파일: `workers/api/src/routes/dashboard.ts`

```
GET  /api/dashboard/shops/:id/coupon  → coupon_config JSON 반환
PUT  /api/dashboard/shops/:id/coupon  → coupon_config 저장 + KV 캐시 무효화
```

`PUT` 요청 구조:
```json
{
  "enabled": true,
  "coupon_no": "12345",
  "coupon_name": "가입 환영 쿠폰",
  "issue_type": "auto",
  "expire_days": 30
}
```

#### 3.2.2 백엔드 — 카페24 쿠폰 API 연동

파일: `packages/cafe24-client/src/index.ts`

추가 메서드:
- `issueCoupon(mallId, memberId, couponNo)` — 특정 회원에게 쿠폰 발급
- `getCoupons(mallId)` — 쿠폰 목록 조회 (대시보드 선택용)

파일: `workers/api/src/routes/oauth.ts`

`/oauth/callback/:provider` 흐름에 쿠폰 발급 훅 추가:
```
신규 가입 감지 (shop_users 신규 생성 시)
  → shop.coupon_config.enabled 확인
  → platform_member_id 확인
  → cafe24Client.issueCoupon() 호출
  → 발급 성공 시 redirect_uri에 bg_coupon=1 파라미터 추가
```

> 주의: 카페24 멤버 ID는 카페24 SSO 완료 후에야 확정됨. 위젯에서 `bg_coupon=1` URL 파라미터를 읽어 가입 완료 페이지에서 쿠폰 알림 팝업 표시.

#### 3.2.3 위젯 — 가입 완료 시 쿠폰 알림

  showCouponNotice(config.coupon_config);

(function() {
  function loadConfig(clientId) {
    return fetch(BASE_URL + '/api/widget/config?client_id=' + clientId)
      .then(function(r) { return r.json(); });
  }

  function loadModule(src) {
    var s = document.createElement('script');
    s.src = src;
    document.head.appendChild(s);
  }

  var clientId = /* ScriptTag src의 쿼리 파라미터에서 추출 */;

  loadConfig(clientId).then(function(config) {
    window.__BG_CONFIG__ = config;
    // 항상 로드: 소셜 로그인 버튼
    loadModule(BASE_URL + '/widget/social-login.js');

    if (config.plan === 'plus') {
      loadModule(BASE_URL + '/widget/mini-banner.js');
      loadModule(BASE_URL + '/widget/exit-popup.js');
      loadModule(BASE_URL + '/widget/escalation.js');
    }
  });
})();

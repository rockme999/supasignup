## 1. 서비스 개요

### 1.1 미션

> **"회원을 모은다"**

번개가입은 쇼핑몰의 비회원을 회원으로 전환하는 데 집중하는 B2B SaaS 서비스입니다.  
소셜 로그인 9종을 무료로 제공하여 진입 장벽을 없애고, 가입 유도 도구(배너/팝업/쿠폰/에스컬레이션)로 회원 유치를 자동화합니다.

**비유**: 번개가입은 **매장 앞 직원**입니다. "어서오세요! 회원가입하면 3,000원 할인이에요!"라고 말하며 방문자를 회원으로 전환시킵니다. 매장 안에서 구매를 유도하는 것은 **otj-shop(오팀장)** 의 영역입니다.

### 1.2 v1과의 차이 (핵심 변경사항)

| 항목 | v1 | v2 |
|------|----|----|
| **미션** | 소셜 로그인 통합 | **"회원을 모은다"** — 가입 전환 집중 |
| **가격** | 무료(월 100명 제한) / 월 29,900원 / 연 329,900원 | **Free(0원) / Plus(월 6,900원 또는 연 79,000원)** |
| **무료 범위** | 월 100명 제한 소셜 로그인 | 소셜 로그인 9종 **무제한** + 가입 쿠폰 1종 |
| **Plus 기능** | 없음 (유료 = 무제한) | 배너/팝업/AI 4종/카카오채널/에스컬레이션 |
| **서비스 경계** | 소셜 로그인 통합만 | 번개가입(유입) vs otj-shop(운영) 역할 분리 |

### 1.3 otj-shop과의 역할 구분

```
[쇼핑몰 방문자]
      ↓
[번개가입] ← 비회원 → 회원 전환 담당
      ↓ (회원 확보 완료)
[otj-shop] ← 회원 → 구매 전환 담당
```

| 서비스 | 역할 | 집중 지표 |
|--------|------|----------|
| **번개가입** | 비회원 → 회원 전환 (유입 퍼널) | 가입 전환율, 가입자 수 |
| **otj-shop** | 회원 → 구매 전환 + 운영 전반 | 구매 전환율, 객단가, 재구매율 |

번개가입은 otj-shop의 **사용자 유입 퍼널** 역할을 하며, 무료 사용자를 확보하여 Plus 유료 전환 후 otj-shop으로 업셀하는 구조입니다.

## 2. 서비스 구성

### 2.1 Free 플랜 (0원, 현재 무료 제공)

Free 플랜은 소셜 로그인 자체를 무료화하여 설치 장벽을 제거하는 것이 목적입니다. 경쟁사 대비 가격 우위와 기능 범위 모두에서 앞서며, 무료임에도 실질적인 가입 전환 효과를 제공합니다.

#### 소셜 로그인 9종 (무제한) ✅ 구현완료

카페24 SSO 기반 소셜 로그인을 운영자 쇼핑몰에 자동 삽입합니다.

| 프로바이더 | 방식 | 한국 MAU | 획득 정보 |
|-----------|------|---------|---------|
| Google | 카페24 SSO 등록 | ~4,000만+ | 이메일, 이름, 프로필사진 |
| 카카오 | 카페24 SSO 등록 | ~4,400만 | 이메일, 이름, 프로필사진, (동의 시) 전화번호/주소 |
| 네이버 | 카페24 SSO 등록 | ~4,600만 | 이메일, 이름, 프로필사진, (동의 시) 전화번호/생일 |
| Apple | 카페24 SSO 등록 | ~1,500만+ | 이메일(숨김 가능), 이름 |
| Discord | ScriptTag 위젯 | ~300만+ | 사용자명, 이메일 |
| Facebook | ScriptTag 위젯 | ~1,000만+ | 이름, 이메일 |
| LINE | ScriptTag 위젯 | ~300만+ | 이름, 프로필사진 |
| X (Twitter) | ScriptTag 위젯 | ~500만+ | 사용자명 |
| Telegram | ScriptTag 위젯 | ~200만+ | 이름, 사용자명 |

> 기술 메모: 카페24 SSO는 최대 5개 항목 등록 가능. Google/Kakao/Naver/Apple 4개를 SSO 등록하고, 나머지 5개(Discord, Facebook, LINE, X, Telegram)는 ScriptTag 위젯 방식으로 처리.

#### 스마트 버튼 ✅ 구현완료

- 가입/로그인 성공 시 `localStorage`에 `bg_last_provider` 저장
- 재방문 시 이전 사용 방법 버튼을 상단 강조 표시 (⚡ 마크 + 테두리 하이라이트)
- 멀티 쇼핑몰 기억: 번개가입을 사용하는 모든 쇼핑몰에서 기억 유지

```
┌──────────────────────────────────────┐
│  ⚡ Google로 로그인 (지난번 사용)       │  ← 강조 (이전 사용 방법)
├──────────────────────────────────────┤
│  카카오로 가입                         │
├──────────────────────────────────────┤
│  네이버로 가입                         │
│  ... (9종 전체 표시)                  │
│                                      │
│  Powered by 번개가입 ⚡              │  ← Free에만 표시
└──────────────────────────────────────┘
```

#### 가입 즉시 쿠폰 자동 발급 (1종) 📋 미구현

- 소셜 가입 완료 즉시 쿠폰 1종 자동 발급
- 운영자가 대시보드에서 쿠폰 금액/조건 설정
- 카페24 쿠폰 API 연동으로 즉시 발급
- Free 제한: 쿠폰 1종만 설정 가능

#### 기본 통계 대시보드 ✅ 구현완료

- 오늘/이번 달 신규 가입 수
- 프로바이더별 가입 수 (Google: N명, Kakao: N명, ...)
- 일별 가입 추이 (30일)
- 플랜 상태 및 사용량

#### "Powered by 번개가입" 브랜딩 ✅ 구현완료

- Free 플랜에서는 위젯 하단에 브랜딩 문구 표시
- Plus 전환 시 완전 제거 가능

### 2.2 Plus 플랜 (월 6,900원 / 연 79,000원)

Plus는 소셜 로그인 이후 **"더 많은 사람을 가입시키고 싶다"**는 니즈를 가진 운영자를 위한 플랜입니다. 가입 유도 도구 + AI 자동화로 운영자의 수작업을 없앱니다.

#### 가입 유도 미니 배너 📋 미구현

- 로그인/회원가입 페이지 상단 또는 하단에 고정 배너 표시
- 예시: "회원가입하면 3,000원 할인! → [지금 가입하기]"
- 운영자가 문구, 색상, CTA 버튼 커스터마이징 가능
- 슬라이드 카드 형태: 혜택 카드 최대 4종 표시

#### 이탈 감지 → 가입 팝업 📋 미구현

- PC: `mouseout` 이벤트 감지 (마우스가 화면 위쪽으로 이탈 시)
- 모바일: `scroll-up` 이벤트 감지 (빠른 위로 스크롤 = 뒤로가기 의도)
- 이탈 직전 팝업: "잠깐! 가입하고 3,000원 받아가세요"
- 쿠폰과 연동하여 즉각적인 혜택 강조
- 쿠키 기반 팝업 노출 제어 (하루 1회 등 설정 가능)

#### 재방문 비회원 에스컬레이션 📋 미구현

비회원이 재방문할 때마다 점점 강도 높은 가입 유도:

| 방문 횟수 | 노출 방식 | 예시 메시지 |
|----------|----------|------------|
| 1~2회 | 작은 배너 (무시 가능) | "회원이 되면 더 많은 혜택이 있어요" |
| 3~4회 | 중간 배너 + 쿠폰 강조 | "3,000원 쿠폰이 기다리고 있어요!" |
| 5회 이상 | 풀스크린 팝업 (1회/방문) | "오늘만 5,000원 쿠폰! 지금 가입하지 않으면 사라져요" |

- 쿠키로 방문 횟수 추적 (localStorage 보완)
- 운영자가 강도 단계별 메시지/조건 커스터마이징 가능

#### 카카오 채널 연동 📋 미구현

- 소셜 가입 완료 시 카카오 채널 자동 친구 추가 (동의 시)
- 운영자의 카카오 비즈채널과 연동
- 카카오 채널 친구 수 증가 → 마케팅 메시지 발송 채널 확보

#### 가입 전환 퍼널 통계 📋 미구현

- 배너 노출 → 클릭 → 소셜 인증 시작 → 가입 완료 → 쿠폰 발급의 단계별 전환율
- 이탈 감지 팝업 노출 수 / 팝업에서 가입 전환율
- 에스컬레이션 방문 횟수별 전환율
- 프로바이더별 가입 완료율 비교

#### "Powered by 번개가입" 브랜딩 제거 📋 미구현

- Plus 전환 시 위젯에서 브랜딩 문구 완전 제거
- 운영자 쇼핑몰 전용 위젯으로 표시

### 2.3 AI 기능 4종 (Plus 전용)

**AI 엔진**: Cloudflare Workers AI — Kimi K2.5 (Moonshot AI)  
256k 컨텍스트, Function calling, Vision 지원. 무료 할당: 일 10,000 Neurons.

#### (1) 쇼핑몰 정체성 자동 설정 📋 미구현

앱 설치 시 1회 실행. AI가 쇼핑몰 URL/상품 데이터를 분석하여 업종, 타겟 고객, 톤앤매너를 자동 파악합니다.

- 예: "여성 패션 쇼핑몰, 20~30대 타겟, 트렌디하고 친근한 톤"
- 이후 모든 카피/전략 생성의 기반으로 활용
- 운영자가 AI 설정 결과를 확인하고 수정 가능

엔드포인트: `POST /api/ai/identity`

#### (2) 주간 AI 브리핑 (매주 1회 자동 생성) 📋 미구현

지난주 성과 분석 + 이번 주 전략을 하나의 리포트로 통합 제공합니다.

```
번개가입 주간 브리핑 (4/1 ~ 4/7)

[지난주 성과]
- 신규 가입 47명 (전주 대비 +12%)
- 카카오 28명 | 구글 11명 | 네이버 8명
- 미니배너 클릭률 14.2% → 가입 전환 8명
- 이탈 팝업에서 5명 전환

[이번 주 전략]
- 봄 신상품 입고 시즌 → "신상품 알림 받기" 카피로 변경 권장
- 네이버 가입 저조 → 버튼 순서를 카카오/네이버/구글로 조정 제안

[AI 추천 액션]
- 미니배너 카피: "봄 신상 10% 할인, 회원만 가능!"
- 이탈 팝업: "지금 가입하면 무료배송 쿠폰 즉시 지급"
```

엔드포인트: `GET /api/ai/briefing`

#### (3) 맥락 기반 가입 유도 카피 생성 (월 10회) 📋 미구현

미니 배너, 이탈 팝업 카피를 쇼핑몰 톤에 맞게 AI가 생성합니다.

- 페이지별 맥락 반영: 상품 상세 → "이 상품 3,000원 할인받고 구매하기"
- 계절/이벤트 자동 반영
- 운영자가 '🤖 AI로 생성' 클릭 → 카피 3~5개 제안 → 선택만 하면 배너/팝업에 즉시 적용
- 월 10회 생성 (주간 브리핑에서 자동 제안 + 수동 요청)

예시 생성 카피:
```
"지금 가입하면 첫 구매 15% 할인 + 5,000원 쿠폰!"
"카카오로 3초 가입 → 즉시 3,000원 적립!"
"회원 전용 특가 상품, 지금 가입하고 확인하세요"
```

엔드포인트: `POST /api/ai/copy`

#### (4) 재방문 비회원 맞춤 메시지 자동 생성 📋 미구현

에스컬레이션 단계별 메시지를 AI가 쇼핑몰 톤에 맞게 자동 생성합니다.

- 1회차: 부드러운 안내 → 3회차: "벌써 3번째 방문이에요! 회원이 되면..."
- 쇼핑몰 정체성 기반 자연스러운 톤 유지
- 운영자가 생성된 메시지 확인/수정 가능

엔드포인트: `POST /api/ai/escalation-copy`

## 3. 사용자 플로우

### 3.1 쇼핑몰 운영자 플로우

#### A. 앱 설치 (카페24 앱스토어) ✅ 구현완료

```
1. 운영자가 카페24 앱스토어에서 "번개가입" 검색/발견
2. "설치하기" 클릭 → HMAC 검증 → 카페24 OAuth 인증 화면 표시
3. 인증 완료 → platform_access_token 발급 (암호화 저장)
4. 번개가입 서버 자동 처리:
   - mall_id, 쇼핑몰 정보 D1에 저장
   - ScriptTag API 호출 → 위젯 JS 자동 등록
   - owner 계정 자동 생성 (이메일: {mall_id}@cafe24.auto)
5. 번개가입 대시보드로 리다이렉트 (쇼핑몰 설정 화면)
```

#### B. 초기 설정 (SSO 연동) ✅ 구현완료

```
1. 대시보드 → 쇼핑몰 설정 → SSO 설정 가이드 진입
2. 운영자가 활성화할 소셜 프로바이더 선택 (기본: Google, 카카오, 네이버, Apple 4개)
3. 쇼핑몰별 client_id / client_secret 자동 생성 + 복사 버튼 제공
4. SSO 설정 가이드 (단계별 스크린샷 포함):
   - Authorize URL: bg.suparain.kr/oauth/authorize?provider={provider}
   - Token URL: bg.suparain.kr/oauth/token
   - UserInfo URL: bg.suparain.kr/oauth/userinfo
5. 운영자가 카페24 관리자 → SSO 설정에서 등록 완료
6. "연동 확인" 클릭 → sso_configured = 1 저장
```

## 8. otj-shop 연동 전략

### 8.1 퍼널 역할 분담

| 단계 | 담당 서비스 | 주요 기능 |
|------|-----------|----------|
| 비회원 방문 → 가입 | **번개가입** | 소셜 로그인, 배너, 팝업, 에스컬레이션 |
| 가입 완료 → 첫 구매 | **번개가입** (쿠폰) + **otj-shop** | 가입 쿠폰 → 즉시 구매 동기 생성 |
| 재방문 회원 → 구매 | **otj-shop** | AI 상품 추천, 타이밍 최적화, CRM |
| 운영 전반 | **otj-shop** | 주문/CS/상품 관리, AI 운영 |

### 8.2 업셀 시나리오

```
1단계: Free 설치
"소셜 로그인 9종 무료로 설치"
→ 가입자 수 증가 확인

2단계: Plus 전환 (월 6,900원)
"배너/팝업으로 가입률을 더 높이고 싶다"
→ 월 6,900원으로 Plus 전환
→ AI 카피 생성으로 추가 전환율 향상

3단계: otj-shop 업셀
"이제 회원 N명이 생겼는데, 이들에게 어떻게 팔지?"
→ otj-shop 소개: 회원 → 구매 전환, AI 운영, 재구매 전략
```

### 8.3 업셀 포인트 (번개가입 대시보드에서 노출)

- 가입자 누적 100명 달성 시: "이제 100명의 회원이 있습니다. otj-shop으로 구매를 유도해보세요" CTA
- 주간 AI 브리핑 하단: "→ 매출 증진 AI 분석은 오팀장(otj-shop)에서" 링크
- Plus 전환 후 온보딩: otj-shop 서비스 소개 카드 표시

### 8.4 번개가입 범위 외 (otj-shop 영역)

번개가입은 "회원 유치"에만 집중하며, 아래는 포함하지 않습니다:

- AI 최적화 (타이밍, 배치, A/B 테스트) — Phase 3 이후 재검토
- 고객 세그먼트/행동 분석 (구매 데이터 기반)
- 구매 촉진 효과 (할인 뱃지, 가격 카운트다운, 재고 긴급)
- 장바구니 회수/재참여 기능
- 주문/CS/상품 관리

### 8.5 기술 연동 구조 (Phase 4 예정)

- otj-shop이 번개가입의 ScriptTag를 원격 제어하는 연동 구조
- 번개가입의 가입 전환 데이터 → otj-shop AI 운영 제안 입력값으로 활용
- **별도 ScriptTag로 분리 운영** (번개가입 ScriptTag ≠ otj-shop ScriptTag)

## 10. 배포 계획

### 10.1 배포 도구

### 10.2 배포 프로세스

```
[개발]
  코드 수정 → 로컬 테스트 → git commit & push
                ↓
[스테이징 배포]
  ./scripts/deploy.sh staging
  ├── D1 마이그레이션 적용 (bg-dev)
  └── Worker 배포 (bg-api-dev → bg-dev.suparain.kr)
                ↓
[스테이징 테스트]
  테스트 쇼핑몰(suparain.cafe24.com)에서 E2E 테스트
  체크리스트 기반 기능 확인
                ↓
[프로덕션 배포]
  ./scripts/deploy.sh production
  ├── D1 백업 (bg-production)
  ├── D1 마이그레이션 적용 (bg-production)
  ├── 스키마 비교 검증 (스테이징 ↔ 프로덕션)
  └── Worker 배포 (bg-api → bg.suparain.kr)
                ↓
[프로덕션 검증]
  health check → 소셜 로그인 스모크 테스트 → 대시보드 접속 확인
                ↓
[문제 발생 시 롤백]
  D1 백업 복원 + 이전 Worker 버전 재배포
```

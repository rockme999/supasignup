## 3. 기존 회원-소셜 계정 연결

### 시나리오
1. 기존 이메일/비밀번호 회원이 소셜 로그인 시도
2. 이메일 매칭 → "기존 회원입니다. 계정을 연결하시겠습니까?" 안내
3. 기존 비밀번호 인증 또는 이메일 인증 후 소셜 계정 연결
4. 이후 소셜 로그인으로 기존 회원 계정에 접근 가능

### 구현 포인트
- `shop_users` 테이블에 동일 이메일 기존 회원 존재 여부 확인 (email_hash 활용)
- 계정 연결 UI (중간 페이지 또는 카페24 연동)
- 1 회원 → N 소셜 계정 연결 지원 (멀티 프로바이더)
- 카페24 `MemberAction.snsLogin` 플로우와의 통합 방법 검토

## 4. 대시보드 앱 개선

### 현재 상태 (V1)
- 홈 (통계 요약)
- 쇼핑몰 관리 (목록, 상세, 프로바이더, SSO 가이드)
- 통합 통계 (필터, 차트)
- 플랜/과금 (결제 연동)
- 계정 설정

### V2 개선 사항
- [ ] 실시간 알림 (한도 근접, 결제 완료 등)
- [ ] 프로바이더별 전환율 분석 (가입 클릭 → 완료 비율)
- [ ] A/B 테스트 (버튼 순서, 스타일 변형)
- [ ] 위젯 커스터마이징 (색상, 위치, 크기 조절)
- [ ] 쇼핑몰별 사용자 목록/검색
- [ ] CSV 내보내기 (통계, 사용자 목록)
- [ ] 구독 만료 알림 + 자동 갱신 처리
- [ ] 모바일 반응형 네비게이션 (햄버거 메뉴)

## 5. 관리자 앱 (신규)

### 목적
- 서비스 전체 운영 관리 (번개가입 운영팀용)
- 일반 사용자 대시보드와 분리

### 기능
- [ ] 전체 쇼핑몰 현황 (등록 수, 활성 수, 플랜 분포)
- [ ] 전체 사용자 통계 (소셜 가입 총 수, 프로바이더 분포)
- [ ] 쇼핑몰 개별 관리 (플랜 변경, 정지, 삭제)
- [ ] 결제 내역 조회/관리 (구독 상태, 환불 처리)
- [ ] 시스템 모니터링 (API 응답 시간, 에러율)
- [ ] 공지사항/알림 관리 (운영자에게 전달)
- [ ] OAuth 프로바이더 상태 모니터링 (키 만료, API 장애)

### 기술 고려
- 별도 라우트 (`/admin/*`) 또는 별도 Worker
- 슈퍼 관리자 인증 (2FA 검토)
- 감사 로그 (누가 무엇을 변경했는지 기록)

[서비스 소개]

"번개가입"은 카페24 쇼핑몰에 소셜 로그인 기능을 제공하는 SSO 앱입니다.

카페24에서 기본 제공하는 소셜 로그인(카카오, 네이버, 구글, 애플)에 더해 Discord, Telegram, Facebook, X(트위터), LINE 등 카페24에서 지원하지 않는 소셜 로그인까지 지원하여 쇼핑몰 고객에게 더 다양한 로그인 수단을 제공하고자 합니다.

현재 저희 서비스의 목표는 기존 아이디/비밀번호 회원과 소셜 로그인을 통합하여, 기존 회원이 소셜 계정을 연동하면 이후 소셜 로그인 시에도 기존 회원 계정으로 자연스럽게 로그인되는 환경을 만드는 것입니다.

그러나 현재 SSO로 로그인하면 기존 회원과 별개로 새 @s 회원이 생성되어, 같은 고객이 2개의 회원 계정을 갖게 되는 문제가 있습니다. 이 문제를 해결하기 위해 아래 API에 대해 문의드립니다.

[문의 1: Customers Social API의 SSO 지원 범위]

GET /api/v2/admin/customers/{member_id}/social

위 API의 응답 예시에서 social_name: "line" 등 카페24 자체 소셜 로그인 연동 정보가 포함되는 것을 확인했습니다.

이 API가 다음 중 어디까지 지원하는지 확인 부탁드립니다:

A. 카페24 자체 소셜 로그인(카카오, 네이버, 구글, 애플 등)으로 연동된 회원 정보만 조회 가능
B. SSO 외부 연동 서비스(저희 "번개가입" 같은 SSO 앱)를 통해 생성된 회원(@s 아이디)의 연동 정보도 조회 가능

[문의 2: Customers Social API 사용 신청]

위 API가 "특정 클라이언트만 사용할 수 있는 API"로 되어 있어 현재 403 응답을 받고 있습니다. 문의 1의 결과와 무관하게 해당 API 사용을 신청합니다.

[문의 3: 기존 회원에 소셜(SSO) 연동 정보를 기록하는 방법]

기존 아이디/비밀번호 회원(예: user1)에 SSO 연동 정보를 기록(추가)할 수 있는 API나 방법이 있는지 문의드립니다.

예를 들어:
- PUT/POST /api/v2/admin/customers/{member_id}/social 같은 쓰기 API
- 또는 기존 회원의 social_member_code를 설정하는 방법
- 또는 기존 회원에 SSO 연동을 등록하는 다른 방법

이것이 가능하면 기존 회원(user1)이 마이페이지에서 소셜 계정을 연동할 때 해당 회원에 SSO 연동 정보를 기록하고, 이후 소셜 로그인 시 새 @s 회원이 생성되지 않고 기존 회원(user1)으로 바로 로그인되게 할 수 있습니다.

이를 통해 카페24 자체 소셜 로그인과 저희 SSO 서비스가 통합되어 쇼핑몰 운영자와 고객 모두에게 더 나은 경험을 제공할 수 있을 것으로 기대하고 있습니다.

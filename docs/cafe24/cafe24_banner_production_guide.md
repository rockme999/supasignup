## 2. 카피(텍스트) 전달 규칙 ★ 반드시 숙지

> **메인 카피와 서브 카피는 이미지에 포함시키지 말 것.**

- 이미지에는 좌측 상단(약 `y=0~300`, `x=130~`) 영역을 **공백**으로 비워 놓는다
- 카피는 **별도 메모/문서**에 기재하여 전달
- 카페24 서버가 **흰색 텍스트로 실시간 합성**해 노출
- 배경이 어두우면 그대로 OK, 밝은 배경이면 합성 후 텍스트가 안 보일 수 있으니 좌측 상단은 **반드시 어두운 색조** 유지

### 전달 샘플
```
[파일] key_visual_1852x840.png
[메인 카피] QR 한 번, 포장 영상 자동 녹화
[서브 카피] 배송 클레임·반품 분쟁도 영상 한 편이면 끝입니다
```

## 3. 타사 배너 벤치마크 — 핵심 공통 패턴

카페24 스토어에서 실제 노출되는 타사 히트 배너를 분석한 결과 **공통된 3가지 패턴**이 발견됨.

### 패턴 1: 단일 강력 포커스
- 우측에 **하나의 지배적 시각 요소** 배치 (거대 숫자, 3D 캐릭터, 제품 목업, 큰 아이콘)
- 보조 요소는 크기를 죽이고 중앙 1개에 시선 집중

### 패턴 2: 좌측 타이틀-우측 비주얼 이원 구조
- 좌측: 메인/서브 카피(카페24 합성)가 들어갈 공간
- 우측: 아이콘/목업/숫자 등 브랜드를 즉각 연상시키는 그래픽
- 중앙 분할선을 넘나드는 요소는 최대 1~2개만

### 패턴 3: 컨트라스트가 강한 배경색
- 다크 네이비, 딥 퍼플, 진한 그라디언트가 주력
- 메인 카피 흰색 합성 가정 → **좌측 상단 영역은 반드시 어두워야** 함

## 4. 디자인 원칙 (VuDrop 제작에서 도출)

### 4-1. 허위/추정 수치는 절대 금지
- 과거 배너에 "93% 클레임 해결률" 같은 근거 없는 수치가 들어가 있었음
- 배너는 광고물이지만 **검증된 사실만** 기재해야 신뢰 훼손 방지
- 허용되는 수치 예: 공식 요금제 스펙(무료 10건, 보관 35일), 제품 동작 사실(QR 스캔 3초)
- 금지되는 수치 예: 측정되지 않은 효과(감소율, 해결률), 시계열 차트로 암시되는 "성과"

### 4-2. 축소 노출(50%)에서도 읽히는 크기
- 제작은 1852×840이지만 **실제 노출은 926×420**
- 작은 뱃지, 얇은 글씨는 축소 후 뭉개짐
- 히어로 요소(숫자, 아이콘, 타이포)는 **최소 200px 이상** 권장
- 본문 텍스트는 22px 이상

### 4-3. 하단 영역 20% 비우기
- 카페24가 페이지 인디케이터, 재생 버튼을 하단에 덮음
- 하단 `y=720~840` 영역은 **로고/브랜드 칩 제외하고는 비워 두기**

### 4-4. 한 줄 가로 뱃지 나열 금지
- 뱃지 3개를 한 줄에 가로 나열하면 926px 축소 시 겹치고 혼잡
- **세로 체크리스트** 또는 **최대 2개 뱃지**만 허용
- 각 항목은 아이콘 + 메인 라인 + 서브 라인 구조로 정보 위계 명확히

### 4-5. 브랜드 칩 위치 선택지
| 위치 | 장점 | 단점 |
|---|---|---|
| 하단 좌측 (full size) | 안정적, 로고/태그라인 명확 | 하단 인디케이터와 간섭 우려 |
| 상단 좌측 (compact) | 카피 영역과 공존 가능 | 카페24 합성 텍스트와 겹칠 수 있음 |
| 우측 비주얼에 통합 | 미려 | 브랜드 인지도 약화 |

VuDrop은 **하단 좌측 full size**를 최종안으로 선택, 상단 좌측 compact 버전도 대안으로 보관.

## 5. 제작 워크플로우 — HTML + Chrome Headless

복잡한 디자인 툴(Photoshop/Figma) 없이도 **HTML/CSS로 제작 → PNG 추출**이 빠르고 버전 관리가 용이.

### 5-1. 폴더 구조
```
docs/cafe24-banners/
├── 1852-840 제작 가이드.png        # 카페24 공식 가이드 원본
├── vudrop-icon.png                 # 브랜드 아이콘
├── banner-concept-{n}.png          # 최종 납품본
├── drafts/
│   ├── concept-{n}.html            # 시안 (scale-wrap으로 50% 축소 미리보기)
│   ├── concept-{n}.png             # 시안 렌더 결과
│   ├── render-{n}.html             # 네이티브 1852×840 렌더용
│   └── final-{n}.html              # 고스트 텍스트 제거된 최종 렌더용
└── archive/                        # 구 버전/참고 자료
```

### 5-2. HTML 템플릿 핵심 구조
```html
<div class="scale-wrap">   <!-- 미리보기용 50% 축소 -->
  <div class="banner">     <!-- 1852×840 -->
    <div class="text-ghost"> ... 좌측 상단 텍스트 자리(반투명) ... </div>
    <div class="hero"> ... 우측 메인 비주얼 ... </div>
    <div class="brand-chip"> ... 로고/브랜드 ... </div>
    <div class="feature-list"> ... 보조 정보 ... </div>
  </div>
</div>
```

### 5-3. Chrome Headless로 PNG 추출
```bash
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
"$CHROME" --headless=new --disable-gpu --hide-scrollbars --no-sandbox \
  --window-size=1852,840 \
  --default-background-color=00000000 \
  --screenshot="banner-concept-b.png" \
  "file://$(pwd)/drafts/final-b.html"
```

### 5-4. 최종 납품본 전처리 (Python 스니펫)
```python
# 1) scale-wrap 제거 (네이티브 1852×840 렌더)
s = s.replace('transform: scale(0.5); transform-origin: top left; ', '')
s = s.replace('<div class="scale-wrap">', '')

# 2) body 배경/패딩 제거
s = s.replace(
    'background: #222; ... padding: 24px;',
    'background: transparent; margin:0; padding:0;'
)

# 3) 텍스트 고스트(미리보기용) 제거
ghost_block = '<div class="text-ghost"> ... </div>'
s = s.replace(ghost_block, '<!-- removed for final -->')
```

## 8. 다음 프로젝트에서 재사용 시 Tips

1. **타사 배너 먼저 캡처**: 카페24 스토어 접속 → 히트 섹션 캡처 → `archive/others-*.png`로 저장
2. **컨셉 3안 기본**: 빅 임팩트 스탯 / 제품 프리뷰 / 히어로 브랜드 — 이 3가지 방향을 항상 먼저 스케치
3. **허위 수치 검증**: 배너에 들어가는 모든 숫자는 `docs/` 내 공식 스펙 파일에서 교차 확인
4. **한 줄 카피 실험**: 메인 카피는 "{주어}, {해결}" 구조가 가장 강함 (예: "배송 클레임, 영상으로 끝")
5. **브랜드 칩 2버전**: 하단 full + 상단 compact 2버전을 항상 함께 제작 (카페24 합성 결과에 따라 폴백 선택)

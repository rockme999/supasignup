# 카페24 Admin API — 수집 가능 데이터 전체 정리

> **작성일**: 2026-04-20 00:16 KST
> **목적**: 번개가입 OAuth 스코프 13종으로 카페24 Admin API에서 가져올 수 있는 데이터 전체 카탈로그.
>          향후 "쇼핑몰 인사이트 DB" 설계 및 다른 카페24 앱/타 프로젝트에서의 재활용 검토용 기초 자료.
> **배경**: 2026-04-19 카페24 담당자 미팅 — 앱 설치 시 동의 거친 범위 내에서 백그라운드 수집·활용 가능 확인.
> **주의**: 본 문서는 *수집 가능성* 카탈로그이며, 실제 수집 항목·주기·저장처는 별도 명세서에서 결정.

## 1. 🟢 메타 정보 (안전, 적극 활용)

| 스코프 | 엔드포인트 | 가져올 정보 | 비고 |
|---|---|---|---|
| `mall.read_store` | `/admin/store` | 쇼핑몰명, 도메인, 사업자등록번호, 대표자, 통신판매업 신고번호, 영업시간, 통화, 시간대, 개설일, 카테고리(업종), 결제수단 설정, 운영자 이메일·전화 | **핵심 메타. 가입 시 1회 수집**. 사업자번호로 업종 세분화 가능 |
| `mall.read_store` | `/admin/shops` | 멀티샵 구성 (한 mall에 여러 shop 운영 여부, 언어/통화별) | 글로벌 진출 여부 판단 |
| `mall.read_application` | `/admin/scripttags` | 설치된 다른 앱의 ScriptTag 목록 | 경쟁앱/연동앱 파악 |

## 2. 🟢 상품/카테고리 (안전, 강력 추천)

| 스코프 | 엔드포인트 | 가져올 정보 | 활용 |
|---|---|---|---|
| `mall.read_product` | `/admin/products` | 상품 목록 (이름, 가격, 공급가, 옵션, 이미지, 카테고리, 등록일, 진열 여부, 판매 여부, 재고) + `/admin/products/count` | **총 상품수, 가격 분포, 평균 객단가, 신상품 등록 패턴** |
| `mall.read_category` | `/admin/categories` | 상품분류 트리 (depth, 분류명, 분류별 상품수) | **업종 정밀 추정**, 카테고리 깊이로 쇼핑몰 규모 |
| `mall.read_collection` | `/admin/brands`, `/admin/manufacturers`, `/admin/trends`, `/admin/classifications` | 취급 브랜드, 제조사, 자체분류, 트렌드 | **편집숍 vs 자사몰 판단**, 브랜드 다양성 |
| `mall.read_supply` | `/admin/suppliers` | 공급사 목록 (수, 거래 형태) | B2B/위탁/직매입 비중 |

→ 인사이트 DB로 **절대 옮기지 말 것**. 다른 프로젝트 공유 금지.

## 9. 정리 — 인사이트 수집 후보 (위험도 0)

수집해도 **법적·윤리적으로 안전**한 그룹만 추리면:

1. **쇼핑몰 메타** — `store`, `shops`, `scripttags`
2. **상품 구성** — `products`(집계), `categories`, `brands`, `suppliers`
3. **매출/방문 통계** — `salesreport` 전체 (이미 카페24가 집계해놓은 비식별 데이터)
4. **회원 집계** — `count`, `customergroups` (등급 구성)
5. **운영 정책** — `coupons`, `benefits`, `shippingcompanies`
6. **인기도** — `wishlists`/`carts` 상품 단위 집계

이 정도면 "쇼핑몰 종합 프로파일"이 완성되고 다른 프로젝트에서도 안전하게 재활용 가능.

## 10. 다음 단계 후보 (별도 명세서에서 확정)

1. **수집 항목 선별** — 위 6개 그룹 중 우선순위 결정 (MVP/Phase2)
2. **수집 주기 설계** — 설치 시 1회 / 일 1회 cron / 주 1회 cron / 웹훅
3. **저장처 결정** — 옵션 비교
   - (A) supasignup 내부 D1에 신규 테이블
   - (B) 책방(MariaDB SUPAKBDB) 옆에 신규 DB(`SUPAINSIGHT` 가칭) 별도 구축
   - (C) 별도 Supabase/D1 신규 프로젝트
4. **스키마 설계** — 플랫폼 확장(아임웹, 식스샵 등) 대비 `platform` 컬럼 분리
5. **법적 검토**
   - 약관/개인정보처리방침 개정 (비식별 집계 데이터를 서비스 개선/리서치에 활용 명시)
   - 카페24 앱 심사 시 데이터 활용 범위 신고
6. **공용 클라이언트 라이브러리** — 다른 프로젝트가 동일 방식으로 read 가능하도록 SDK화

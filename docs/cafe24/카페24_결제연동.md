# 카페24 인앱결제 연동

> 작성일: 2026-03-18 KST
> 참조: supaMov24, VeriPack-Multi 프로젝트의 카페24 결제 구현

## 결제 플로우

```
1. 대시보드 /billing → "월간 전환" 클릭
2. POST /api/dashboard/billing/subscribe  {plan, shop_id}
   → subscriptions 테이블에 pending 레코드 생성
   → Cafe24 API: POST /api/v2/admin/appstore/orders (주문 생성)
   → confirmation_url 반환
3. 프론트: window.open(confirmation_url) → 카페24 결제 페이지
4. 결제 완료 → return_url 콜백 (팝업 닫기만)
5. 카페24 웹훅 event_no=90157 → 실제 플랜 전환
   → subscription.status = active
   → shop.plan = monthly/yearly
6. 환불 시 → 웹훅 event_no=90159
   → subscription.status = cancelled
   → shop.plan = free
```

## Cafe24 API 스펙

### 주문 생성
```
POST https://{mall_id}.cafe24api.com/api/v2/admin/appstore/orders
Authorization: Bearer {access_token}
Content-Type: application/json

{
  "request": {
    "order_name": "번개가입 월간 플랜",
    "order_amount": "29900",
    "return_url": "https://bg.suparain.kr/api/dashboard/billing/return",
    "automatic_payment": "F"
  }
}

→ { "order": { "order_id": "...", "confirmation_url": "..." } }
```

### 필요 Scope
- `mall.write_application` (주문 생성)
- `mall.read_application` (주문 조회) — 이미 install 시 요청됨

## 3. wrangler.toml 구조

```toml
# ── 공통 설정 ──
name = "bg-api"
main = "src/index.ts"
compatibility_date = "2026-03-01"

[triggers]
crons = ["0 * * * *"]

# ── 프로덕션 (기본, npx wrangler deploy) ──
[[routes]]
pattern = "bg.suparain.kr"
custom_domain = true

[vars]
BASE_URL = "https://bg.suparain.kr"

[[d1_databases]]
binding = "DB"
database_name = "bg-production"
database_id = "254438d7-b1b1-45a1-afe1-2c766aba252b"
migrations_dir = "migrations"

[[kv_namespaces]]
binding = "KV"
id = "fe38c181822b4729850b53ab839339b5"

# ── 스테이징 (npx wrangler deploy --env dev) ──
[env.dev]
name = "bg-api-dev"

[[env.dev.routes]]
pattern = "bg-dev.suparain.kr"
custom_domain = true

[env.dev.vars]
BASE_URL = "https://bg-dev.suparain.kr"

[[env.dev.d1_databases]]
binding = "DB"
database_name = "bg-dev"
database_id = "{새로 생성 후 입력}"
migrations_dir = "migrations"

[[env.dev.kv_namespaces]]
binding = "KV"
id = "{새로 생성 후 입력}"
```

## 4. 배포 명령어

```bash
# 스테이징 배포
npx wrangler deploy --env dev

# 프로덕션 배포
npx wrangler deploy

# 스테이징 DB 마이그레이션
npx wrangler d1 migrations apply bg-dev --env dev

# 프로덕션 DB 마이그레이션
npx wrangler d1 migrations apply bg-production

# 스테이징 시크릿 등록
npx wrangler secret put ENCRYPTION_KEY --env dev

# 프로덕션 시크릿 등록
npx wrangler secret put ENCRYPTION_KEY
```

## 8. 개발 워크플로우

```
[기능 개발]
  코드 작성
    ↓
  npx wrangler deploy --env dev  (스테이징 배포)
    ↓
  suparain999에서 테스트
    ↓
  문제 없으면
    ↓
  npx wrangler deploy  (프로덕션 배포)
    ↓
  실제 고객 쇼핑몰 확인

[DB 마이그레이션]
  마이그레이션 SQL 작성
    ↓
  npx wrangler d1 migrations apply bg-dev --env dev
    ↓
  스테이징에서 테스트
    ↓
  npx wrangler d1 migrations apply bg-production
```

## 9. 구축 절차 (순서)

```
1. Cloudflare D1 데이터베이스 생성 (bg-dev)
2. Cloudflare KV 네임스페이스 생성 (bg-dev-kv)
3. wrangler.toml에 [env.dev] 블록 추가
4. 스테이징 DB 마이그레이션 적용
5. 스테이징 Secrets 등록
6. 스테이징 Worker 배포 + 도메인 확인
7. 카페24 개발자센터에서 "번개가입 (개발용)" 앱 생성
8. 스테이징 앱에 suparain999 설치
9. 소셜 프로바이더에 스테이징 redirect_uri 추가
10. 전체 플로우 테스트
```

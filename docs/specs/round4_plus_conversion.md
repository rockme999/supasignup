## 1. 배경 — 라운드 1~3에서 도달한 결론

### 1-1. 진단

- **Plus 기능들은 거의 모두 구현되어 있다** — 하지만 *운영자가 가치를 체감하지 못해* 결제 안 함.
- 라운드 2까지의 안(손실 회피 카드, 임계점 알림, 라이브 카운터)은 모두 *운영자에게 말로 설득*하는 방식. 그러나 운영자는 말로 안 설득됨.
- **운영자 본인의 손가락이 갈 곳에 막힌 게 있어야** 결제한다. *내적 욕구*에서 결제 동기가 시작되는 패턴이 결정적 (Notion / Figma / Canva의 freemium hook).
- 동시에 운영자가 *매일 대시보드에 들어올 이유*가 없으면 retention이 약해진다.

### 1-2. 라운드 4의 핵심 패러다임

> **"Plus 결제 전환"은 *운영자에게 파는 것*이 아니라, *운영자 스스로 욕구를 발견하게* 하는 것.**

3개 트랙으로 정리:

| 트랙 | 본질 | 1주차 | 2주차+ |
|---|---|---|---|
| **A. 가입 전환 실데이터 생성** | "Plus가 가입자를 늘린다"를 사실로 만듦 | — | Exit-intent 쿠폰 / Smart trigger / 라이브 카운터 |
| **B. 운영자 인지·감정 자극** | 운영자가 *자기 욕구*에서 결제 동기 발견 | **Plus 디자인 프리셋 preview** ⭐ | 손실 회피 카드 / 임계점 알림 |
| **C. 운영자 retention** | 매일 대시보드 들어올 이유 | **What's New 인디케이터** | AI 주간 브리핑 이메일 |

**1주차는 B + C를 동시에 깐다.** A는 2주차 이후 (실데이터 누적 후 손실 회피 카드 의미가 살아남).

## 2. 1주차 3개 작업 단위

### W1-1. 로그인 디자인 Plus 프리셋 (preview 패턴) ⭐ 메인

**현재 상태**:
- Free 사용자도 5종 디자인 프리셋(컬러 / 모노톤 / 테두리 / 호버채움 / 아이콘만) 무제한 사용
- 슬라이더 6종으로 너비·높이·간격·둥글기·아이콘 간격·왼쪽 여백 자유 조정

**추가**:
- Plus 전용 디자인 프리셋 **5~10종** (하단 §3에서 후보 정의)
- Free 사용자도 Plus 프리셋을 *클릭해서 미리보기 + 슬라이더 조작* 가능
- **저장(`[디자인 저장]` 버튼) 클릭 시** Plus 업그레이드 모달 노출
- Plus 사용자는 모달 없이 즉시 저장

**핵심 원칙**:
- *체험은 끝까지 자유, 영구 저장만 잠금*
- 잠금 모달 ❌, 시도 자체를 막는 UI ❌
- Plus 프리셋 카드에 작은 💎 배지로 *유료임을 미리 알림* (자물쇠 아님)
- 워터마크 X — 운영자가 미리보기에서 진짜 적용된 모습을 볼 수 있어야 욕구가 생김

**사용자 시각 흐름**:
1. /dashboard/login-design 진입
2. 프리셋 영역에 Free 5종 + Plus 5~10종 모두 표시
3. Plus 프리셋 클릭 → 미리보기 즉시 적용 (실시간)
4. 슬라이더 조작 → 미리보기 갱신
5. 마음에 들면 `[디자인 저장]` 클릭
6. 결제 모달 노출 (W1-3에서 정의)
7. 결제 완료 또는 취소 → 적절히 fallback

### W1-2. What's New 인디케이터

**목적**: 운영자가 매일 대시보드에 들어와서 *변화를 발견*하는 retention.

**구성**:
- 좌측 사이드바 메뉴 항목별 빨간 점 / NEW 배지
- 대상 메뉴 후보:
  - `로그인 디자인` — Plus 프리셋 신규 추가 시
  - `AI 보고서` — 새 주간 브리핑 생성 시
  - `변경사항` (사이드바 푸터) — 새 버전 배포 시 (PUBLIC_CHANGELOG 업데이트)
- 메뉴 진입 시 자동 해제

**데이터**:
- KV 키: `seen:{owner_id}:{key}` 값: ISO 시각 (마지막 본 시점)
- 비교 대상:
  - `seen:{owner_id}:design-presets` vs Plus 프리셋 마지막 추가 시각
  - `seen:{owner_id}:changelog` vs PUBLIC_CHANGELOG 빌드 시각 (build-info.ts의 BUILD_TIME)
  - `seen:{owner_id}:ai-briefing` vs 최신 ai_briefings.created_at

**구현 단순화**:
- 1주차에선 `changelog` + `design-presets` 2개만 (AI 브리핑 인디케이터는 별도 후속)
- KV 호출은 layout.tsx에서 한 번에 (배치)

### W1-3. 결제 유도 모달

**노출 시점**: W1-1에서 Plus 프리셋이 적용된 상태로 `[디자인 저장]` 클릭 시.

**모달 카피** (메모리 `project_pricing_strategy` 반영):

```
┌─────────────────────────────────────┐
│  💎 Plus로 업그레이드하면           │
│  이 디자인이 그대로 적용됩니다       │
│                                     │
│  ○ 월 6,900원                       │
│    매달 부담 없이 시작               │
│                                     │
│  ● 연 79,000원                      │
│    결제 공백 없이 1년 안심 운영      │
│    비슷한 가입 유도 앱들의           │
│    월 비용 수준의 연간 가격          │
│                                     │
│  [Plus 시작하기]    [나중에]        │
└─────────────────────────────────────┘
```

**행동**:
- `[Plus 시작하기]` 클릭 → 카페24 인앱결제 흐름 (이미 구현된 `billing.tsx` 활용)
- `[나중에]` 클릭 → 모달 닫힘. 미리보기는 *그대로 유지* (운영자가 다음에 다시 시도할 수 있도록)
- 결제 완료 시 자동으로 디자인 적용 + "Plus 활성화 완료" 토스트

**디자인 톤**:
- 화이트 심플 (대시보드 톤 유지)
- 보라/인디고 그라디언트는 헤딩에만 (스토리 #3 톤)
- 비교 표 톤(월간/연간 라디오)으로 운영자가 *선택권*을 느끼게

## 3. Plus 디자인 프리셋 후보 5~10종

각 프리셋은 *Free와 명확히 구별되는 시각 매력* + *Plus 다움*을 가져야 함.

| # | 이름 | 컨셉 | 핵심 시각 | Plus 다움 |
|---|---|---|---|---|
| 1 | **글래스모피즘** | 반투명 유리 | backdrop-filter blur + 반투명 배경 | 트렌디, 고급감 |
| 2 | **네온 글로우** | 빛나는 테두리 | box-shadow neon + 다크 모드 친화 | 비비드, 클럽/패션 |
| 3 | **그라디언트 플로우** | 풍부한 색상 전환 | linear/radial gradient 5색 | 화려함, 의류/뷰티 |
| 4 | **소프트 섀도우** | 부드러운 부유감 | 다층 box-shadow + 라운드 | 안정감, 가구/리빙 |
| 5 | **3D 입체** | 깊이감 + 클릭 효과 | transform translateY + shadow | 게임/엔터 |
| 6 | **펄스 애니메이션** | 미세한 움직임 | @keyframes pulse 1.5s | 클릭 유도 강 |
| 7 | **미니멀 라인** | 가는 선 + 타이포 | 1px border + 큰 폰트 | 미니멀, 잡화 |
| 8 | **카드 스타일** | 입체 카드 | 두꺼운 border + 위로 떠오름 | 식음료/리뷰 |
| 9 | **브랜드 매칭** ✨ | AI가 쇼핑몰 분위기 추출 | shop_identity 기반 자동 추천 | AI 차별화 |
| 10 | **2025 트렌드** | 비비드 + 큰 라운드 | 풀 라운드 + 비비드 단색 | 시즌 감각 |

**1주차 출시 — 6종 (사용자 합의)**: #1 글래스모피즘, #2 네온 글로우, **#11 리퀴드 글래스 (신규)**, #3 그라디언트 플로우, #4 소프트 섀도우, #6 펄스 애니메이션
**2주차 추가 — 5종**: #5 3D 입체, #7 미니멀 라인, #8 카드 스타일, #9 브랜드 매칭 AI, #10 2025 트렌드

**#11 리퀴드 글래스 (신규)** — Apple iOS의 Liquid Glass 인스파이어드. CSS만으론 Apple의 실시간 픽셀 샘플링은 X지만 backdrop-filter blur+saturate + 안쪽 광택 box-shadow + 마우스 위치 추적 동적 그라디언트로 *동적 굴절·광택* 효과. 글래스모피즘(#1)과 분리한 이유: #1은 정적 반투명, #11은 동적 광택.

**디자인 작업 방식 (사용자 합의)**:
AI 시안 생성 → 사용자 큐레이션 사이클. 디자이너 의뢰 X.
- Day 1: AI가 6종 HTML/CSS 시안 작성 → `docs/specs/round4_preview/index.html` (정적 미리보기)
- Day 2: 사용자가 브라우저에서 직접 보고 피드백 ("색 바꾸자", "이건 빼자" 등) → 반복 큐레이션 → 최종 6종 확정
- Day 3 이후: 정식 코드 통합

## 4. 데이터 모델 / 마이그레이션

### 4-1. shops.widget_style JSON 확장

기존:
```json
{ "preset": "default", "buttonWidth": 280, "buttonGap": 8, "borderRadius": 10, "align": "center" }
```

추가 필드 — Plus 프리셋 식별자:
```json
{
  "preset": "glassmorphism",   // Plus 프리셋 ID도 같은 필드
  "presetTier": "plus",         // "free" | "plus" — 적용 시점에 검증
  "buttonWidth": 280,
  ...
}
```

**구현**: 마이그레이션 불필요. JSON 컬럼이라 자유 확장. 백엔드에서 `presetTier === 'plus'` 일 때 shops.plan === 'plus' 검증.

### 4-2. KV — What's New 인디케이터

키 패턴: `seen:{owner_id}:{key}`
값: ISO 시각 (예: `"2026-04-27T11:25:46Z"`)

조회 시점: layout.tsx에서 dashboard 진입 시 한 번 (3~4 KV.get 묶어서)
갱신 시점: 운영자가 해당 메뉴 진입 시 비동기로 KV.put

**구현**: KV 사용 (이미 인프라 있음). 별도 마이그레이션 불필요.

## 6. 성공 지표

라운드 4 1주차 *효과 측정용 지표* (funnel_events 활용):

| 지표 | 의미 | 목표 (1주 누적) |
|---|---|---|
| Plus 프리셋 시도율 | Free 사용자 중 Plus 프리셋을 *클릭한* 비율 | 20% 이상 |
| 저장 → 결제 모달 노출 | Plus 프리셋 시도 후 저장 클릭 비율 | 30% 이상 |
| 결제 모달 → 결제 완료 | 모달 노출 → 실제 결제 비율 | 5% 이상 (적당함) |
| 결제 전환 절대 수 | 1주 누적 신규 Plus 결제 | 베이스라인 + α |

지표 자체는 별도 funnel_events 이벤트 추가 필요:
- `widget_style.preview_plus_preset` (Plus 프리셋 클릭)
- `widget_style.save_attempt_locked` (저장 시 모달 노출)
- `billing.upgrade_modal_shown`
- `billing.upgrade_completed_via_design_preview`

이건 W1-1 구현에 함께 박음.

## 1. 서비스 포지셔닝

### 1.1 한 줄 정의

**"카페24 쇼핑몰 무료 9종 소셜 로그인 + ScriptTag 기반 부가 서비스 플랫폼"**

### 1.2 핵심 메시지

> "무료로 시작, 부가기능으로 매출 성장"

- 소셜 로그인 9종은 **영구 무료** -- 진입 장벽 제거
- 부가기능은 **후불제, 기능별 ON/OFF** -- 쓴 만큼만 과금, 끄면 0원
- 쇼핑몰 운영자에게 "무료니까 일단 설치해보세요"라고 말할 수 있는 구조

### 1.3 경쟁사 대비 차별점

| 차별 요소 | 경쟁사 현황 | 번개가입 |
|-----------|------------|---------|
| 소셜 프로바이더 | 1~4종 (카카오 중심) | **9종** (Google, Kakao, Naver, Apple, Discord, Facebook, LINE, X, Telegram) |
| MZ 대응 | 없음 (카카오 채널만 집중) | **Discord 로그인 + 커뮤니티 연동** |
| 해외 소셜 | 미지원 | Google, Apple, Discord, LINE, X, Telegram |
| 기본 가격 | 무료~월 69,900원 | **소셜 로그인 완전 무료 (영구)** |
| 확장 모델 | 고정 월정액 | 부가기능 후불제 (ON/OFF) |
| AI 활용 | 미활용 | **Cloudflare Workers AI 기반 스마트 배너/A/B 최적화** |
| 플랫폼 확장성 | 카페24 전용 | 카페24 집중, 멀티 플랫폼은 향후 검토 |

## 2. 수익 모델

### 2.1 무료 기본 (소셜 로그인 9종)

| 항목 | 내용 |
|------|------|
| 범위 | Google, Kakao, Naver, Apple, Discord, Facebook, LINE, X, Telegram |
| 가격 | **0원 (영구 무료)** |
| 제한 | 없음 -- 쇼핑몰당 가입 수 무제한 |
| 포함 기능 | 소셜 로그인, 스마트 버튼, 마이페이지 연동 위젯, 기본 통계 |

> **가격 전략 변경 사항**: PRD v1에서는 무료 플랜을 "월 100명 제한"으로 설정했으나, 경쟁사 분석 결과 알파푸시가 무료 모델로 6개월 만에 6,000개 쇼핑몰을 확보한 점을 감안하여 소셜 로그인 자체는 완전 무료로 전환한다. 수익은 부가기능에서 창출한다.

### 2.2 부가기능 (후불제, 기능별 ON/OFF)

**원칙**: 기능을 켜면 사용량 기반 과금, 끄면 과금 없음. 서비스 이전 부담 최소화.

> ScriptTag가 이미 쇼핑몰 전 페이지에 삽입되어 있으므로, 모든 부가기능은 추가 설치 없이 ON만으로 활성화 가능하다.

#### Phase 1 (MVP 동시 출시)

| 부가기능 | 설명 | 과금 방식 |
|---------|------|----------|
| **미니 배너 (하단 혜택 배너)** | 1초가입 스타일 슬라이드 카드 배너 — 로그인 폼 아래 혜택 유도 카드 4종 ("다양한 혜택을 받으세요", "혜택은 오직 회원만" 등). **AI 배너 카피라이터 포함**: 운영자가 '🤖 AI로 생성' 버튼 클릭 → Kimi K2.5가 쇼핑몰 상품/이벤트 기반 카피 3~5개 제안 → 선택만 하면 완료 | 기본 1개 무료, 복수 배너 월정액 |
| **카카오 채널 연동** | 가입 시 카카오 채널 자동 친구 추가 — 운영자 핵심 요구 기능, 경쟁사 4곳 모두 제공 | 월정액 |

#### Phase 2 (출시 후 1~2개월)

| 부가기능 | 설명 | 과금 방식 |
|---------|------|----------|
| **소셜 프루프 위젯** | "방금 N명이 카카오로 가입했어요" 실시간 알림 (ScriptTag) | 월정액 |
| **이탈 감지 + 소셜 가입 팝업** | 마우스 이탈/뒤로가기 감지 시 소셜 가입 유도 팝업 | 월정액 |
| **프로바이더별 전환 분석 대시보드** | 프로바이더별 클릭->가입 전환율, 일별/주별 추이, 이탈 지점 분석 | 월정액 |

#### Phase 3 (차별화 -- AI 활용)

| 부가기능 | 설명 | 과금 방식 |
|---------|------|----------|
| **AI 스마트 배너** | Cloudflare Workers AI (Kimi K2.5)로 배너 카피/이미지(FLUX) 자동 생성 — 상품/이벤트에 맞는 문구 자동 제안 | 월정액 |
| **AI A/B 테스트 최적화** | 버튼 순서, 배너 문구 자동 최적화 — 데이터 축적이 곧 lock-in | 월정액 |
| **AI 고객 세그먼트 분석** | 소셜 프로바이더별 구매 전환율, 객단가 분석 — 운영자에게 인사이트 제공 | 월정액 |
| **supa24 Discord 커뮤니티 연동** | Discord 로그인 시 브랜드 서버 자동 초대 — MZ 킬러 기능 | 월정액 |

#### Phase 4 (생태계)

| 부가기능 | 설명 | 과금 방식 |
|---------|------|----------|
| **수파레인 크레딧 쿠폰 시스템** | 번개가입 소셜 가입 시 생태계 서비스(supaMov, VuDrop 등) 체험 크레딧 자동 발급 | 연동 무료 |
| **VuDrop 포장 영상 알림** | ScriptTag으로 주문 포장 동영상 등록 시 고객 알림/링크 공유 | 월정액 |
| **오팀장 AI 운영 연동** | 소셜 전환 분석 데이터를 오팀장에 전달, AI 기반 운영 제안 생성 | 향후 확정 |

### 2.3 이탈 비용 설계

서비스 이전 자체는 쉽게 만들되, **데이터 축적이 자연스러운 이탈 비용**을 만든다.

```
[설치 직후]
  - 진입 비용: 0원 (무료)
  - 이탈 비용: 낮음 (아직 데이터 없음)

[3개월 후]
  - 전환 분석 데이터 축적
  - 프로바이더별 성과 비교 가능
  - 이탈 비용: 중간 (분석 데이터 유실)

[6개월 후]
  - A/B 테스트 최적화 결과 축적
  - 소셜 프루프 전환율 데이터
  - 이탈 비용: 높음 (최적화 히스토리 전체 유실)
```

### 2.4 AI 활용 전략 (Cloudflare Workers AI)

경쟁사(1초가입, 알파푸시 등)는 아직 AI를 본격 활용하지 않고 있다. Cloudflare Workers AI 무료 티어(10,000 Neurons/일)를 활용하여 선점한다.

#### MVP에 포함하는 AI

| AI 기능 | 구현 방식 | Phase | 근거 |
|---------|---------|:-----:|------|
| **AI 배너 카피라이터 (미니 배너 통합)** | Kimi K2.5 1회 호출, 프롬프트 엔지니어링만으로 구현. 별도 기능이 아닌 미니 배너의 UX 개선 | **Phase 1 (MVP)** | 경쟁사 전체 AI 미활용 → 이것 하나만으로 차별점. 데이터 축적 불필요. |

#### MVP에 넣지 않는 AI (데이터 축적 필요)

| AI 기능 | 이유 | 전환 시점 |
|---------|------|---------|
| **AI 전환율 인사이트** | 3개월 이상 데이터 축적 필요 | Phase 3 |
| **AI A/B 테스트 최적화** | 충분한 트래픽과 배너 노출 데이터 필요 | Phase 3 |
| **AI 고객 세그먼트 분석** | 구매 전환 데이터 축적 필요 | Phase 3 |

#### 범위 밖 (본업 이탈)

| AI 기능 | 이유 |
|---------|------|
| AI 상품 추천 위젯 | 크리마/알파푸시 영역 |
| AI 챗봇 | 채널톡 영역 |
| AI 가격 최적화 | 오팀장 영역 |
| AI 이미지 생성 (FLUX) | 품질 일관성 보장 어려움 |
| 감정 분석 (distilbert) | 리뷰 데이터 접근 필요, 크리마 영역 |

#### 가입 유도 타이밍

| Phase | 방식 |
|:-----:|------|
| MVP | 규칙 기반 — 상품 N개 조회 후, 체류 N분 후 등 |
| Phase 3 | AI 전환 — ScriptTag 행동 데이터 기반 최적 타이밍 예측 |

#### 전체 AI 모델 로드맵

| AI 모델 | 용도 | Phase |
|---------|------|:-----:|
| **Kimi K2.5** (256k 컨텍스트, tool calling, 비전) | **배너 카피라이터 (MVP 미니 배너 통합)**, 고객 행동 분석, 세그먼트 분석 | **Phase 1** / Phase 3 |
| **FLUX** (이미지 생성) | 배너 이미지 자동 생성 — 범위 밖으로 결정 | 범위 밖 |
| **distilbert** (감정 분석) | 리뷰/댓글 자동 분류 — 범위 밖으로 결정 | 범위 밖 |

**AI가 만드는 lock-in 효과:**

```
[설치 직후]
  - AI 없음 → 이탈 비용 낮음

[3개월 후 — AI A/B 테스트 활성화]
  - 버튼 순서, 배너 문구 최적화 데이터 축적
  - "어떤 조합이 전환율 최고인지" AI가 학습
  - 이탈 시 최적화 히스토리 전체 유실

[6개월 후 — AI 세그먼트 분석 활성화]
  - 프로바이더별 구매 전환율, 객단가 인사이트
  - 맞춤형 배너 자동 생성 히스토리
  - 이탈 비용: 매우 높음 (대체 불가한 데이터 자산)
```

> **비용 구조**: 무료 10,000 Neurons/일, 초과 시 $0.011/1,000 Neurons. 쇼핑몰 1곳당 일일 AI 사용량이 극소하므로 수백 개 쇼핑몰까지 무료 티어 내에서 운영 가능.

### 2.5 1초가입 "로그인 페이지 대체" 전략 대응

1초가입은 로그인 페이지를 완전히 대체하는 자체 UI를 제공한다 (특허 출원번호 30-2020-0039257). 이에 대한 대응:

| 대응 항목 | 전략 | Phase |
|-----------|------|:-----:|
| **하단 혜택 배너** | 경쟁 필수 — 슬라이드 카드 형태로 반드시 구현 | Phase 1 |
| **로그인 페이지 커스터마이징** | 테마 기능 검토 — 브랜드 이미지 배경, 레이아웃 변경 | Phase 2~3 |

## 3. 시장 진입 전략

### 3.1 4단계 진입 로드맵

**1단계: 유인 -- 무료 9종 소셜 로그인**

- "카카오만 쓰고 계세요? 9종 소셜 로그인이 무료입니다"
- 경쟁사 대비 소셜 프로바이더 수 압도 (1~4종 vs 9종)
- 해외 고객이 있는 쇼핑몰 타겟 ("Google, Apple 로그인 지원하세요?")
- 앱스토어 리뷰/설치 수 초기 확보에 집중

**2단계: 정착 -- 데이터 축적**

- 전환 분석 대시보드 제공 (무료 기본 통계 -> 유료 상세 분석)
- "어떤 소셜이 전환율이 높은지" 인사이트 제공
- 데이터가 쌓일수록 이전 동기 감소

**3단계: 전환 -- 부가기능 ON**

- 데이터 기반으로 "미니 배너를 켜면 전환율 N% 상승 예상" 제안
- 자연스러운 유료 전환 (강제 아닌 데이터 기반 제안)
- ON/OFF 자유 -> 시도 부담 최소

**4단계: 고착 -- 경쟁사 이전 동기 제거**

- 축적된 전환 데이터 + A/B 테스트 최적화 결과
- 수파레인 생태계 연동 (supa24, 크레딧 등)
- 경쟁사에 없는 고유 기능 (Discord 커뮤니티, 소셜 프루프 등)

### 3.2 경쟁사 고객 전환 시나리오

경쟁사에서 번개가입으로의 전환은 카페24 플랫폼 특성상 "공존 -> 점진적 전환"이 핵심이다.

| 전환 유형 | 난이도 | 전략 |
|-----------|:------:|------|
| 소셜 없는 쇼핑몰 | 쉬움 | 바로 설치, 기존 ID/PW 회원은 SNS 연동 설정으로 자동 매핑 |
| 카페24 자체 소셜 (@g/@n/@k) | 보통 | 번개가입 SSO 추가 + 기존 소셜 버튼 CSS 숨김 |
| 경쟁 SSO (1초가입 등) | 보통 | SSO 2번 슬롯에 번개가입 추가 (@s + @s1 공존) |

> 상세 전환 가이드: `소셜연동_설계.md` 섹션 6 참조

## 4. 수파레인 생태계 연동

### 4.1 확정된 연동

#### supa24 (브랜드 플레이그라운드 Discord)

| 항목 | 내용 |
|------|------|
| 연동 방식 | Discord 로그인 -> 브랜드 Discord 서버 자동 초대/참여 |
| 대상 | 번개가입 Discord 로그인을 활성화한 쇼핑몰 |
| 가치 | MZ 타겟 킬러 조합 -- 가입과 동시에 커뮤니티 진입 |
| 구현 시점 | MVP Phase 2 |

```
[쇼핑몰 방문자]
  1. Discord로 소셜 가입
  2. 번개가입이 Discord Bot API로 브랜드 서버 초대
  3. 가입 완료 + 커뮤니티 참여 동시 달성
```

이것은 **번개가입 + supa24 조합으로만 가능**한 기능이다. 경쟁사는 Discord 로그인 자체를 지원하지 않으므로 복제 불가.

#### 오팀장 (AI 쇼핑몰 운영)

| 항목 | 내용 |
|------|------|
| 연동 방식 | 소셜 전환 분석 데이터를 오팀장에 전달, AI 기반 운영 제안 생성 |
| 대상 | 번개가입 + 오팀장 동시 사용 쇼핑몰 |
| 가치 | "어떤 소셜이 매출 기여도가 높은지" AI 분석 |
| 구현 시점 | 오팀장 런칭 이후 접점 개발 (향후 검토) |

### 4.2 크레딧 연동 (간접 시너지)

| 항목 | 내용 |
|------|------|
| 구조 | 번개가입으로 가입 -> 수파레인 생태계 서비스 크레딧 쿠폰 발급 |
| 대상 서비스 | supaMov, VuDrop 등 수파레인 유료 서비스 |
| 가치 | 번개가입이 생태계 유입 모태 역할 (무료이므로 진입 장벽 없음) |
| 구현 시점 | 향후 검토 -- 생태계 서비스 사용자 규모에 따라 결정 |

```
[번개가입 소셜 가입]
  -> 수파레인 크레딧 쿠폰 자동 발급 (ScriptTag 기반)
  -> supaMov: 동영상 인코딩 크레딧
  -> VuDrop: 포장 동영상 저장 크레딧
  -> 생태계 전체 체험 유도
```

### 4.3 향후 검토 (서비스 접점)

아래 연동은 방향성은 정해졌으나, 구체적 구현 범위는 각 서비스의 성숙도에 따라 결정한다.

| 서비스 | 연동 아이디어 | 필요 조건 | 상태 |
|--------|-------------|-----------|------|
| **VuDrop** | 주문 포장 동영상 등록 시 ScriptTag로 알림/링크 공유 | VuDrop ScriptTag API 정의 | 향후 검토 |
| **QR2** | QR 스캔 -> 소셜 로그인 -> 즉시 가입 (오프라인 연동) | QR2 리다이렉트 플로우 정의 | 향후 검토 |

## 6. MZ 타겟 전략

### 6.1 핵심 인사이트

```
카카오 채널 = 4050 세대의 커뮤니케이션
Discord     = MZ 세대의 커뮤니케이션
```

경쟁사 4곳 모두 카카오 채널 연동에만 집중하고 있다. Discord를 소셜 로그인으로 지원하는 카페24 앱은 **번개가입이 유일**하다.

### 6.2 Discord 커뮤니티 커머스 모델

```
[기존 — 카카오 채널 모델]
  가입 -> 카카오 채널 친구 추가 -> 알림톡 발송 -> 재방문
  (일방향 커뮤니케이션, 4050 중심)

[번개가입 — Discord 커뮤니티 모델]
  가입 -> Discord 서버 자동 참여 -> 커뮤니티 활동 -> 재방문
  (양방향 커뮤니케이션, MZ 중심)
```

| 비교 항목 | 카카오 채널 | Discord 커뮤니티 |
|-----------|-----------|-----------------|
| 소통 방향 | 일방향 (브랜드->고객) | 양방향 (고객<->고객<->브랜드) |
| 콘텐츠 | 알림톡, 쿠폰 | 실시간 대화, 이벤트, 투표 |
| 주 타겟 | 4050 세대 | MZ 세대 (10~30대) |
| 커뮤니티 형성 | 불가 (1:1 채널) | 가능 (서버 내 채널, 역할, 봇) |
| 이탈 비용 | 낮음 (차단 한 번) | 높음 (커뮤니티 관계) |

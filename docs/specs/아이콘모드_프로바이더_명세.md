> 작성: 2026-04-30 12:41 KST

## 1. 배경 / 목적

### 문제
- 지원 소셜 로그인 프로바이더가 늘어남에 따라(google, kakao, naver, apple, discord, telegram, + 향후 facebook/x/line/toss/tiktok), 모든 프로바이더를 풀버튼으로 노출하면 UI가 길어져 **선택 부담(choice paralysis)** 이 커진다
- 그러나 일부 프로바이더만 노출하면 해당 프로바이더 사용자가 가입 자체를 못 하게 되어 **CR(전환율) 손실** 이 발생한다

### 해결
프로바이더별로 **버튼 모드**(눈에 띄게 보임) 와 **아이콘 모드**(컴팩트하게 노출)를 운영자가 선택할 수 있게 한다.

- 버튼 모드: 풀폭 소셜 버튼 (현재 동작과 동일)
- 아이콘 모드: 풀폭 버튼 영역 아래에 가로 정렬된 아이콘만 (44×44 정도)

운영자는 **자주 쓰이는 카카오/네이버는 버튼**, **나머지는 아이콘** 식으로 운영하여 컴팩트하면서도 모든 프로바이더의 기능은 유지한다.

### 핵심 보존 요구사항
- **last-used override 유지**: 직전에 로그인한 프로바이더는 모드 설정과 무관하게 **풀버튼으로 최상단에 highlight 노출**(현재 `bg-btn-highlight` 동작과 동일). 사용자가 Discord(아이콘 모드)로 마지막 로그인했으면, 다음 방문 시 Discord 풀버튼이 최상단에 노출되어야 한다.

## 2. 데이터 모델

### 2.1 DB 스키마 변경

`shops` 테이블에 컬럼 추가:

```sql
ALTER TABLE shops ADD COLUMN icon_providers TEXT NOT NULL DEFAULT '[]';
```

- 형식: JSON 문자열 — 아이콘 모드인 프로바이더 ID의 배열, 예: `["discord","telegram"]`
- 제약: `icon_providers ⊆ enabled_providers` 가 항상 성립해야 함 (검증은 PUT 엔드포인트에서)
- 기본값 `'[]'` — 기존 샵에 영향 없음 (전부 버튼 모드 = 현재 동작 유지)
- 컬럼명 컨벤션: `enabled_providers` 와 동일한 snake_case (CLAUDE.md 네이밍 일관성 규칙 준수)

### 2.2 마이그레이션 파일

```
migrations/0020_shops_add_icon_providers.sql
```

내용:
```sql
ALTER TABLE shops ADD COLUMN icon_providers TEXT NOT NULL DEFAULT '[]';
```

### 2.3 데이터 흐름

```
어드민 UI (체크박스) ─PUT─▶ /api/dashboard/shops/{id}/providers
                               { providers: [...], icon_providers: [...] }
                               │
                               ├─validate: icon_providers ⊆ providers
                               ├─validate: providers.length ≥ 1
                               └─store: shops 테이블 업데이트

위젯 ─GET─▶ /api/widget/config?client_id=xxx
              ◀─ { providers: [...], icon_providers: [...], style: {...}, plan: ... }
              │
              └─render: providers 를 (button-mode, icon-mode) 로 분리 후 렌더
```

### 2.4 타입 정의 (TypeScript)

```ts
// 기존
type ShopConfig = {
  providers: ProviderName[];     // 활성화된 모든 프로바이더, 순서 보존
  // ... style, plan, ...
};

// 변경 후
type ShopConfig = {
  providers: ProviderName[];          // 동일 — 활성화된 모든 프로바이더 순서 보존
  iconProviders?: ProviderName[];     // 그 중 아이콘 모드인 프로바이더 (subset)
  // ... style, plan, ...
};
```

`iconProviders` 가 빈 배열이거나 미정의면 전부 버튼 모드 (= 기존 동작 동일).

## 3. UI/UX 설계

### 3.1 어드민 설정 화면 (`views/settings.tsx` 프로바이더 영역)

현재 각 프로바이더 row 구성:
```
[토글] [컬러닷] [이름]                    [▲][▼]
```

변경 후:
```
[토글] [컬러닷] [이름]    [버튼/아이콘 토글]    [▲][▼]
```

- 추가 컨트롤: **모드 토글** (라디오 또는 세그먼티드 컨트롤). 디자인 옵션:
  - A. 작은 버튼 2개: `[버튼] [아이콘]` — 명확
  - B. 체크박스: `□ 아이콘으로 표시` — 더 컴팩트
  - **권장: B** (사용자 요구사항이 "체크 박스를 두고")

- 비활성화된 프로바이더(off)면 모드 토글도 자동 비활성화/숨김
- 모든 프로바이더가 아이콘 모드면 버튼 영역이 빌 수 있음 → **최소 1개는 버튼 모드 필수** 검증 (또는 허용? 결정 필요 — **권장: 허용**, 단 UX 경고 노출. "버튼이 하나도 없으면 첫인상이 약합니다" 같은 톤)

### 3.2 어드민 라이브 미리보기 (settings.tsx 내부 JS, 기존 1000~1175 라인 영역)

- 현재 "프로바이더 미리보기"가 풀버튼으로만 그려짐
- 변경 후: 위젯 실제 렌더와 동일한 split (버튼 → 아이콘 row) 적용

### 3.3 실제 위젯 렌더 (`widget/core/render.ts`)

모드별 분리 + 순서:

```
[Last-used 풀버튼 (highlight)] ← 강제 promote (만약 last-used가 icon-mode였어도)
[Button 모드 프로바이더 (last-used 제외)]
[Icon row: 아이콘 모드 프로바이더 가로 정렬]
[powered by 번개가입]
```

- `bg-btn-highlight`(파란 테두리 + "이전에 사용" 라벨) 는 last-used에만 적용 — 기존 동작 유지
- 아이콘 row 는 별도 컨테이너 div: `class="bg-icon-row"` — `display:flex; flex-direction:row; flex-wrap:wrap; justify-content:center; gap:8px; margin-top:8px`
- 아이콘 자체는 기존 `icon-only` preset 의 단일 버튼 스타일을 재사용 (44×44 원형 또는 정사각형, gap, 호버 동일)

## 4. 렌더 로직 상세

### 4.1 sortProviders 확장

현재 `BGWidget.prototype.sortProviders` 는 last-used를 맨 앞에 둠. 그대로 유지하되, 호출 후 결과를 split 한다:

```js
// 의사코드
var ordered = this.sortProviders(this.config.providers);  // last-used가 [0]
var iconSet = new Set(this.config.iconProviders || []);
var lastUsed = this.lastProvider;

var buttonProviders = [];
var iconProviders = [];
for (var i = 0; i < ordered.length; i++) {
  var p = ordered[i];
  if (p === lastUsed) {
    buttonProviders.push(p);  // last-used는 무조건 버튼으로 promote
  } else if (iconSet.has(p)) {
    iconProviders.push(p);
  } else {
    buttonProviders.push(p);
  }
}
```

### 4.2 렌더 분기

```js
// 1. 버튼 영역 — 기존 그대로
for (var i = 0; i < buttonProviders.length; i++) {
  var btn = this.renderButton(buttonProviders[i], i === 0 && this.lastProvider === buttonProviders[i]);
  this.container.appendChild(btn);
}

// 2. 아이콘 row — 새로 추가
if (iconProviders.length > 0) {
  var iconRow = document.createElement('div');
  iconRow.className = 'bg-icon-row';
  for (var j = 0; j < iconProviders.length; j++) {
    var ico = this.renderIconButton(iconProviders[j]);  // ← 새 메서드 또는 기존 renderButton(p) + icon-only 렌더 옵션
    iconRow.appendChild(ico);
  }
  this.container.appendChild(iconRow);
}

// 3. powered by — 기존 그대로
```

### 4.3 renderIconButton 구현 옵션

**옵션 A. 신규 메서드 추가**
- 단점: renderButton과 코드 중복
- 장점: 분기 로직이 깔끔

**옵션 B. renderButton에 `iconOnlyOverride` 파라미터 추가**
- renderButton(provider, isHL, iconOnlyOverride=false) — true 면 preset이 'icon-only'인 것처럼 렌더
- 장점: 코드 재사용
- 단점: 시그니처 확장

**권장: 옵션 B** — 기존 renderButton의 icon-only 분기(line 259~274)를 그대로 재활용.

### 4.4 Plus 프리셋 / preset 상호작용

| preset | 처리 방식 |
|---|---|
| `default`, `mono`, `outline`, `outline-mono` | 정상 split: 버튼 + 아이콘 row |
| `icon-only` | **icon_providers 무시** — 전부 아이콘 (현재 동작 유지). 어드민 UI에서도 모드 토글 비활성화 |
| Plus 6종 (glass/neon/liquid/gradient/soft/pulse) | 정상 split. **단, 아이콘은 Plus 스타일 적용 안 함** (단순 원형/정사각 아이콘만). Plus는 풀버튼 디자인을 위한 프리셋이므로 |

### 4.5 dark-wrap 상호작용

- `glassmorphism` / `neon-glow` / `liquid-glass` 자동 다크 wrapper는 **컨테이너 전체** 에 적용됨
- 즉 다크 wrap 안에 풀버튼 + 아이콘 row 가 모두 들어감
- 아이콘 row 의 아이콘들은 다크 배경 위에 놓이므로 **fill 자동으로 흰색** 적용 (현재 Plus dark fill 로직과 동일)

### 4.6 아이콘 row CSS

```css
.bg-widget .bg-icon-row {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
  justify-content: center;
  align-items: center;
  gap: 8px;
  margin-top: 8px;
  width: 100%;
}
.bg-widget .bg-icon-row .bg-btn {
  /* icon-only preset 스타일 재사용: 44x44, 둥근 모서리, 패딩 0 */
  width: 44px !important;
  height: 44px !important;
}
```

### 4.7 모바일 대응

- 아이콘 row는 `flex-wrap: wrap` 으로 폭이 좁으면 자동 줄바꿈
- 모바일 폰트 사이즈/터치 타겟은 44×44 유지(애플 가이드라인 최소)

## 5. API 변경

### 5.1 PUT `/api/dashboard/shops/:id/providers`

**기존 요청:**
```json
{ "providers": ["google","kakao","naver","apple"] }
```

**변경 후:**
```json
{
  "providers": ["google","kakao","naver","apple","discord","telegram"],
  "icon_providers": ["discord","telegram"]
}
```

**검증:**
- `providers.length >= 1`
- `icon_providers ⊆ providers`
- 각 항목이 알려진 프로바이더인지

**backward compat:** `icon_providers` 미지정 시 빈 배열로 저장.

### 5.2 GET `/api/widget/config?client_id=xxx`

응답에 `iconProviders` 필드 추가:
```json
{
  "providers": [...],
  "iconProviders": [...],
  "style": {...},
  "plan": "..."
}
```

camelCase로 직렬화 (기존 응답 컨벤션 따름).

## 6. Edge Cases / 결정 사항

| 케이스 | 처리 |
|---|---|
| `icon_providers` 가 `providers` 의 부분집합 아님 | API 검증에서 400 반환 |
| 모든 프로바이더가 icon-mode (버튼 0개) | **허용**, 단 어드민에서 경고 toast: "풀버튼이 없으면 첫인상이 약할 수 있습니다" |
| 활성 프로바이더 1개인데 그게 icon-mode | 위와 동일 — 허용 + 경고 |
| `preset === 'icon-only'` | icon_providers 무시 (모두 아이콘). 어드민 UI 에서 모드 토글 비활성화 + 안내 |
| last-used가 icon-mode 인 프로바이더 | last-used만 풀버튼으로 promote (icon row 에서는 제외) |
| last-used가 더 이상 활성 프로바이더가 아님 (운영자가 비활성화) | 현재 sortProviders 가 이 경우 처리 — 그냥 sorting 하지 않고 원본 순서 반환 |
| 프로바이더 순서 변경 (▲▼) | order 는 `providers` 배열 순서. icon row 안에서도 동일 순서 유지 |
| 미사용/비활성화 토글 시 icon_providers에 들어 있던 항목 | 자동 제거 (DB 검증 단계에서) |

- 라이브 카운터/이탈 인텐트 등 다른 위젯 기능과의 시각적 충돌 점검.

## 10. 롤백 계획

이 기능 도입 후 문제 발생 시:
1. **빠른 롤백**: 운영자가 어드민에서 모든 프로바이더를 버튼 모드로 되돌리면 즉시 기존 UI 복원 (DB 데이터만 변경, 코드 변경 없음)
2. **코드 롤백**: 이 기능 구현 커밋 직전 (`b79f64d`) 으로 git revert
3. **DB 롤백**: `icon_providers` 컬럼은 default `'[]'` 라 그냥 두어도 무해 (코드가 읽지 않으면 무시됨). 굳이 컬럼 drop 안 해도 됨.

## 2. Admin API — 검증 완료 항목

### 2.1 인증 (OAuth 2.0)

```
토큰 갱신: POST https://{mall_id}.cafe24api.com/api/v2/oauth/token
  grant_type: refresh_token
  refresh_token: {refresh_token}
  Authorization: Basic base64(client_id:client_secret)

★ 중요: API 호출 시 401 → refresh_token으로 갱신 후 재시도 패턴 필수
★ 중요: X-Cafe24-Api-Version 헤더 반드시 포함 (예: '2026-03-01')
```

**cafe24-common 반영 사항**: `X-Cafe24-Api-Version` 헤더가 일부 API에서 필수. 특히 쿠폰 API는 버전 헤더 없으면 파라미터가 달라질 수 있음.

### 2.2 쇼핑몰 정보 조회 ✅

```
GET /api/v2/admin/store
Scope: mall.read_store

응답 핵심 필드:
  shop_name           쇼핑몰명 ("수파레인")
  company_condition   업종 ("도소매")
  company_line        업태 ("수출")
  mall_url            도메인 ("http://suparain.kr")
  sales_product_categories  판매 카테고리 (["Undecided"] — 대부분 미설정)
  customer_service_phone    고객센터 전화번호
  country_code        국가 코드 ("KOR")

★ sales_product_categories는 대부분 "Undecided" → 업종 추론은 카테고리 트리 분석 필요
```

### 2.3 카테고리 조회 ✅

```
GET /api/v2/admin/categories?limit=50
Scope: mall.read_category

응답 구조:
  category_no         카테고리 번호
  category_name       이름 ("Outerwear", "Tops" 등)
  category_depth      깊이 (1=대분류, 2=중분류, 3=소분류, 4=상세분류)
  parent_category_no  상위 카테고리 번호

실테스트: 19개 카테고리, depth 1~4 확인
```

### 2.4 상품 수 조회 ✅

```
GET /api/v2/admin/products/count
Scope: mall.read_product

응답: { "count": 5 }
```

### 2.5 쿠폰 생성 ✅ (정액 + 정률)

```
POST /api/v2/admin/coupons
Scope: mall.write_promotion
Header: X-Cafe24-Api-Version: 2026-03-01 (필수!)

[정액 할인 (benefit_type: 'A')]
{
  "shop_no": 1,
  "request": {
    "coupon_name": "쿠폰명",
    "benefit_type": "A",
    "discount_amount": {                    // ★ 객체!
      "benefit_price": 3000                 // ★ 정수!
    },
    "discount_rate": null,
    "available_period_type": "F",
    "available_begin_datetime": "2026-04-01T00:00:00+09:00",
    "available_end_datetime": "2026-05-01T23:00:00+09:00",
    "available_site": ["W", "M"],           // ★ 생성시 배열!
    "available_scope": "O",
    "available_amount_type": "E",
    "available_coupon_count_by_order": 1,
    "available_price_type": "U",
    "available_payment_method": ["all"],
    "issue_type": "M",                      // M=대상자지정, A=조건부자동
    "issue_reserved": "F",
    "issue_order_date": "F"
  }
}

[정률 할인 (benefit_type: 'B')]
{
  "request": {
    "benefit_type": "B",
    "discount_amount": null,
    "discount_rate": {                      // ★ 객체!
      "benefit_percentage": 10,             // 10% 할인
      "benefit_percentage_round_unit": "10", // ★ 문자열! (10원 단위 절사)
      "benefit_percentage_max_price": 5000   // 최대 할인금액 (null 가능)
    },
    // ... 나머지 동일
  }
}

[가입 시 자동 발급 쿠폰]
{
  "request": {
    "issue_type": "A",                      // 조건부 자동
    "issue_sub_type": "J",                  // ★ J=가입 시!
    "issue_member_join": "T",               // ★ 필수 (J일 때)
    "issue_member_join_recommend": "F",     // ★ 필수 (J일 때)
    "issue_member_join_type": "N",          // ★ 필수 (J일 때) — N=제한없음
    // ... 나머지 동일
  }
}
```

**cafe24-common 반영 사항**:
- `discount_amount`는 문자열이 아닌 **객체** `{ benefit_price: int }`
- `discount_rate`도 **객체** `{ benefit_percentage: int, benefit_percentage_round_unit: str }`
- `available_site`는 생성 시 **배열**, 조회 시 **쉼표 구분 문자열**
- `benefit_price`는 **정수** (소수점 문자열 불가)
- `issue_sub_type: 'J'` 사용 시 `issue_member_join` 관련 3개 필드 모두 필수

### 2.6 쿠폰 발급 ✅ (전체 + 특정 1명)

```
POST /api/v2/admin/coupons/{coupon_no}/issues
Scope: mall.write_promotion

[전체 회원 발급]
{
  "request": {
    "issued_member_scope": "A"
  }
}

[특정 회원 1명 발급] ← 핵심!
{
  "shop_no": 1,
  "request": {
    "issued_member_scope": "M",
    "member_id": "bbbig",              // ★ 문자열! (배열 아님!)
    "allow_duplication": "T",
    "single_issue_per_once": "T",      // ★ T=1장만 발급
    "send_sms_for_issue": "F"
  }
}
→ 응답: { "issues": { "count": { "coupon_no": 1 } } }
```

**cafe24-common 반영 사항**:
- 특정 회원 발급 시 `member_id`는 **문자열** (기존 `issued_member_id` 배열 아님!)
- `single_issue_per_once: 'T'`면 1장, `'F'`면 `issue_count_per_once`(최소 2)만큼
- `issue_count_per_once` 최소값 **2** (1로 설정하면 422 에러)

### 2.7 쿠폰 상태 변경/삭제 ✅

```
PUT /api/v2/admin/coupons/{coupon_no}
Scope: mall.write_promotion

일시정지: { "request": { "status": "pause", "immediate_issue_pause": "I" } }
재개:     { "request": { "status": "restart", "immediate_issue_restart": "I" } }
삭제:     { "request": { "status": null, "deleted": "D" } }
                                                    // ★ 'D' (T가 아님!)
```

### 2.8 쿠폰 조회 ✅

```
GET /api/v2/admin/coupons
Scope: mall.read_promotion

응답 주의:
  benefit_price: "2999.00"       (소수점 문자열로 반환)
  available_site: "W,M"          (쉼표 구분 문자열로 반환 — 생성시는 배열!)
  deleted: "F"                   (삭제 여부)
  issued_count: "12"             (발급 수)
```

### 3.1 초기화

```javascript
// client_id는 서버에서 ScriptTag JS에 런타임 주입
(function(CAFE24API) {
  // SDK 사용 가능
})(CAFE24API.init({
  client_id: 'D786ohzUHVsjrjGOTdbx0A',
  version: '2022-12-01'
}));
```

### 3.2 상품 정보 조회 ✅

```javascript
CAFE24API.get('/api/v2/products/15', function(err, res) {
  if (!err) {
    // res.product.product_name  → "보이런던 스커트 B63SK03F"
    // res.product.price         → "69000.00"
    // res.product.category      → [{ category_no: 34 }]
    // res.product.product_code  → "P000000P"
```

**실테스트 결과**: 상품명, 가격, 카테고리, 상품코드 전부 조회 성공.

### 3.5 제약사항

| 항목 | 내용 |
|------|------|
| **CORS** | ScriptTag에서 직접 fetch 불가 (401 "not an allowed client_id"). SDK만 사용 |
| **콜백 전용** | Promise/async-await 미지원. `.then()` 호출 시 에러 |
| **심사 주의** | ScriptTag 내에서 Admin API 직접 호출 금지 (앱스토어 심사 탈락) |
| **Rate Limit** | Leaky Bucket 방식 |
| **init 필수** | `CAFE24API.init({ client_id, version })` 없이 호출하면 인증 실패 |

| 함정 | 올바른 방법 |
|------|-----------|
| `discount_amount: "3000"` | ❌ → `{ benefit_price: 3000 }` (객체+정수) |
| `discount_rate: 10` | ❌ → `{ benefit_percentage: 10, benefit_percentage_round_unit: '10' }` (객체) |
| `available_site: "W,M"` | 생성 시 `["W","M"]` (배열), 조회 시 `"W,M"` (쉼표 문자열) |
| `benefit_price: "3000.00"` | ❌ → `3000` (정수) |
| `benefit_percentage_round_unit: 'F'` | ❌ → `'10'` (절사 단위, 정수 문자열) |
| `deleted: 'T'` | ❌ → `'D'` |
| 발급 시 `issued_member_id: ['id']` | ❌ → `member_id: 'id'` (문자열!) |
| `single_issue_per_once: 'F'` + `issue_count_per_once: 1` | ❌ → 최소값 2. 1장이면 `single_issue_per_once: 'T'` |
| `issue_sub_type: 'J'`만 입력 | ❌ → `issue_member_join` + `recommend` + `type` 3개 필수 |
| API 버전 헤더 없이 호출 | ❌ → `X-Cafe24-Api-Version` 필수 |

### 7.2 Front SDK 함정

| 함정 | 올바른 방법 |
|------|-----------|
| `CAFE24API.get(url).then(...)` | ❌ → `CAFE24API.get(url, function(err, res) {...})` 콜백만! |
| `fetch('cafe24api.com/...')` | ❌ 401 → CAFE24API SDK 경유만 가능 |
| ScriptTag에서 Admin API 호출 | ❌ 심사 탈락 → 백엔드에서 처리 |
| `CAFE24API.init('client_id')` | △ → `CAFE24API.init({ client_id: '...', version: '2022-12-01' })` 권장 |

### 7.3 토큰 관리

```
access_token 만료 시 401 → refresh_token으로 갱신 → 재시도 패턴 필수
갱신된 토큰은 DB에 암호화 저장 (AES-GCM)
refresh_token 만료 시 → 앱 재설치(재인증) 필요
```

### 7.4 적립금 (Mileage)

```
카페24에서 적립금은 현금성 자산으로 분류
→ Scope (mall.write_mileage)가 개발자센터에 노출되지 않음 (별도 신청 필요)
→ 번개가입 MVP에서는 쿠폰으로 대체 결정 (2026-04-01)
→ Front SDK getPointInfo()로 적립금 조회는 가능 (mall.read_customer)
→ API 지급이 필요하면 카페24에 별도 신청
```

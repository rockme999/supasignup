## 1. 설계 원칙

### 1.1 단일 쇼핑몰 구조

- 카페24 앱스토어에서 앱을 설치하면 **1 설치 = 1 쇼핑몰**이 자동으로 대시보드에 연결됨
- 사용자가 직접 쇼핑몰을 "등록"하거나 "삭제"하는 UI가 **없음**
- 쇼핑몰 추가/삭제는 카페24 앱스토어에서 앱을 설치/삭제하는 것으로 대체
- 대시보드 접속 시 이미 연결된 쇼핑몰의 데이터를 바로 보여줌

### 1.2 Free/Plus 업셀 UX

- Plus 전용 메뉴는 사이드바에서 `[Plus]` 뱃지와 함께 항상 노출
- 클릭 시 잠금 상태로 진입하되, 잠금 UI 내부에 기능 미리보기와 업그레이드 버튼 표시
- **잠금 UI 패턴**: 흐린 미리보기 배경 + 중앙에 "Plus 전용 기능입니다" + "업그레이드하기" 버튼
- Free 사용자가 Plus 기능을 자주 보게 되어 자연스럽게 업셀 유도

### 1.3 레이아웃 & 디자인 패턴

- **supaMov24 패턴 참고**: 사이드바 네비게이션, 홍보 배너 슬라이더, 요약 카드 3열
- **기술 스택**: Hono JSX (SSR), Tailwind CSS, Alpine.js, HTMX — 변경 없음
- 사이드바: 고정 좌측 네비게이션 (모바일에서는 접힘)
- 컨텐츠 영역: 우측 메인 콘텐츠 (max-width 제한)

## 2. 사용자 대시보드 메뉴 구조

### 2.1 전체 메뉴 트리

```

[메인]
├── 홈                          ← 전체 요약 (오늘 가입 수, 미니 차트, 알림)

[설정] (쇼핑몰 관리자 작업 순서대로 배치)
├── 기본 설정                    ← 쇼핑몰 정보, 쇼핑몰 정체성 확인/수정
├── SSO 설정 가이드              ← 단계별 스크린샷 + 복사 버튼
├── 소셜 프로바이더              ← 9종 토글, 활성화 선택
├── 로그인 디자인                ← 버튼 디자인 프리셋 + 커스터마이징
├── 쿠폰 설정                    ← 가입 쿠폰 설정 (다중은 Plus)
├── 미니배너 [Plus]              ← 배너 카피/위치 설정
├── 이탈 감지 팝업 [Plus]        ← 팝업 카피/쿨다운 설정
├── 에스컬레이션 [Plus]          ← 재방문 단계별 메시지 설정
├── 카카오 채널 [Plus]           ← 채널 ID + 친구 추가 설정
└── AI 설정 [Plus]              ← 정체성 재분석 + AI 카피 생성

[기타]
├── 플랜 & 결제                  ← Free/Plus 전환, 결제 관리
├── 사용 가이드                  ← SSO 설정, FAQ, 기능 안내
├── 문의하기                    ← 1:1 문의 게시판
└── (상단 우측) 프로필/로그아웃
```

### 2.2 상단 홍보 배너

supaMov24의 배너 API를 연동한다. VeriPack-Multi와 동일하게 서버 측 프록시 방식으로 구현한다.

**연동 방식 (서버 측 프록시)**:
```typescript
// Hono 라우트: GET /dashboard/promo-banner
app.get('/dashboard/promo-banner', async (c) => {
  try {
    const resp = await fetch(
      'https://sm.suparain.kr/api/v1/banners/active/html?app=supasignup',
      { signal: AbortSignal.timeout(5000) }
    );
    return c.html(resp.ok ? await resp.text() : '');
  } catch {
    return c.html('');
  }
});
```

**클라이언트 측 삽입 (HTMX)**:
```html
<div id="promo-banner"
     hx-get="/dashboard/promo-banner"
     hx-trigger="load"
     hx-target="this"
     hx-swap="innerHTML">
</div>
```

**동작 규칙**:
- 배너 응답이 비어 있으면 공간 없이 완전 비노출
- 닫기 클릭 시 `localStorage`에 배너 ID 저장 → 재노출 안 함
- 새 세션(로그인) 시 dismiss 초기화 (`?new_session` 파라미터)
- 다중 배너 시 Alpine.js 슬라이더로 5초 자동 전환
- supaMov 어드민에서 `target_app=supasignup` 배너를 추가하면 자동 반영

## 4. 설정 페이지 상세

### 4.1 기본 설정

URL: `/dashboard/settings/general`

| 항목 | 설명 |
|------|------|

**쇼핑몰 정체성 서브 섹션**:
```
┌─────────────────────────────────────────────┐
│  쇼핑몰 정체성 (AI 자동 분석)                  │
│                                             │
│  업종: 여성 패션                              │
│  타겟 고객: 20~30대 여성                      │
│  톤앤매너: 트렌디하고 친근한 톤                  │
│                                             │
│  [수정]  [AI 재분석]  [확정]                 │
│                                             │
│  ⚠ 미확정 상태 — 확정하면 AI 기능이 정확해져요   │
└─────────────────────────────────────────────┘
```

**정체성 확정 플로우**:
1. 앱 설치 시 `POST /api/ai/identity` 백그라운드 자동 호출 → `shop_identity` 저장
2. 미확정 상태면 홈에 알림 카드 표시
3. 운영자가 결과 확인 → (수정 후) "확정" 클릭 → `is_confirmed = true`
4. 확정 후 홈 알림 카드 제거, AI 기능이 정체성 데이터 기반으로 동작

### 4.3 SSO 설정 가이드

URL: `/dashboard/settings/sso-guide`

단계별 스크린샷 포함 가이드 페이지

```
Step 1: 카페24 관리자 → 앱스토어 → SSO 설정 진입
Step 2: 프로바이더 추가 클릭
Step 3: 아래 정보를 복사하여 붙여넣기

  Authorize URL:   [복사] bg.suparain.kr/oauth/authorize?provider=kakao
  Token URL:       [복사] bg.suparain.kr/oauth/token
  UserInfo URL:    [복사] bg.suparain.kr/oauth/userinfo
  Client ID:       [복사] bg_abc123xyz
  Client Secret:   [복사] sk_live_xxxxxx

Step 4: 저장 후 테스트 로그인 진행
Step 5: 아래 "설정 완료" 체크박스 체크
```

- 복사 버튼 클릭 시 클립보드에 복사 + "복사됨!" 피드백
- 하단 "설정 완료 ✓" 체크박스 (체크 시 `sso_configured = true` 저장)
- SSO 미완료 상태면 홈에 별도 안내 없음 (가이드 페이지에만 표시)

| 기준 요소 CSS 셀렉터 | 배너가 표시될 기준 위치 (기본: `#top_nav_box`) |
| 이전 로그인 기록 감지 시 숨김 | 토글 |
| 기본값 되돌리기 | 버튼 |

**기준 요소 위치 동작 (2026-04-05 개선)**:
- `position: fixed`로 뷰포트에 고정, 기준 요소 바로 아래에 플로팅
- 기준 요소가 뷰포트 절반 위로 올라오면 배너 출현 (fadeIn/slideDown 애니메이션)
- 기준 요소가 뷰포트 밖으로 나가면 배너 숨김
- 페이지 로드 후 800ms 딜레이로 출현 효과 보장
- CSS 셀렉터 지원: `#id`, `.class`, `.classA.classB`, 공백 구분 복수 클래스 자동 변환
- 기준 요소가 DOM에서 제거/재생성되면 자동 재탐색 (`document.body.contains` 체크)
- `#tabProduct`를 기준으로 설정 시 sticky 충돌 방지 로직 적용

### 5.5 문의 관리

  - 메타: 제목 / 작성자 email / 쇼핑몰 / 작성일시

- 향후 확장 (미구현): 문의자 쇼핑몰 플랜/가입자 수 요약, closed 상태 전환 UI


| is_confirmed | INTEGER | 0=미확정, 1=확정 |

## 8. 기술 구현 노트

### 8.1 SSR 방식 유지

- Hono JSX (SSR) 방식 그대로 유지
- 사이드바 네비게이션 컴포넌트화 (supaMov24 패턴 참고)
- Alpine.js: 클라이언트 인터랙션 (토글, 모달, 슬라이더)
- HTMX: 부분 갱신 (배너 로드, 폼 저장, 데이터 갱신)

### 8.2 홍보 배너 연동

- 서버 측 프록시 엔드포인트 `GET /dashboard/promo-banner` 추가
- supaMov24의 `sm.suparain.kr/api/v1/banners/active/html?app=supasignup` 호출
- 타임아웃 5초, 예외 시 빈 HTML 반환 (배너 비노출)
- `target_app=supasignup` 으로 필터링 (supaMov 어드민에서 별도 등록 필요)

### 8.3 Plus 잠금 UI 공통 컴포넌트

```tsx
// PlusLockOverlay 컴포넌트 — 재사용
<PlusLockOverlay feature="미니배너">
  {/* 흐린 미리보기 내용 */}
</PlusLockOverlay>
```

내부 구현:
- `is_plus` 플래그 체크
- Plus면 children 그대로 렌더
- Free면 children을 `opacity-30 pointer-events-none`으로 처리 + 오버레이 카드

### 8.4 관리자 콘솔 라우트

기존 `/admin` 라우트 확장:
- `/admin/ai-reports` — AI 보고서 열람 (신규)
- `/admin/inquiries` — 문의 관리 (신규)
- `/admin/announcements` — 공지사항 (신규)
- `/admin/audit-logs` — 감사 로그 (신규)
- `/admin/system` — 시스템 상태 (신규)

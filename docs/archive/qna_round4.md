## 2. 카페24 회원 API 조사 결과 (핵심)

### 결론: "회원 자동 생성은 가능, 자동 로그인은 직접 불가"

#### 가능한 것
- **Admin API로 회원 생성**: `POST /admin/customers`로 외부 OAuth 정보를 이용해 카페24 회원 생성 가능
- **회원 정보 수정**: `PUT /admin/customers/{member_id}`로 나중에 추가 정보 업데이트 가능
- **SSO 연동**: 카페24가 공식 지원하는 SSO 로그인 연동 기능 존재

#### 불가능한 것
- **API만으로 로그인 세션 강제 생성**: 보안상 사용자의 명시적 인증 없이는 불가
- **ScriptTag로 로그인 세션 생성**: Script Tag는 이벤트 감지/정보 전달용일 뿐

## 3. 구현 가능한 시나리오 3가지

### 시나리오 A: SSO 연동 방식 (가장 추천)

```
[사용자] → "구글로 가입" 버튼 클릭
       → Google OAuth 인증 (이미 로그인 상태면 자동)
       → 우리 서버에서 Google 사용자 정보 수신
       → Admin API로 카페24 회원 자동 생성
       → 카페24 SSO 메커니즘으로 로그인 세션 생성
       → 사용자는 쇼핑몰에 로그인된 상태
```

**장점**: 카페24 공식 지원 기능, 로그인 세션이 카페24에서 관리됨
**단점**: 쇼핑몰 관리자가 SSO 설정을 해야 함 (앱 설치 시 자동화 가능한지 조사 필요)
**핵심 질문**: 앱 설치 과정에서 SSO 설정을 API로 자동 활성화할 수 있는가?

### 시나리오 B: Admin API 회원 생성 + 자동 로그인 유도

```
[사용자] → "구글로 가입" 버튼 클릭
       → Google OAuth 인증
       → Admin API로 카페24 회원 생성 (임시 비밀번호)
       → 프론트에서 생성된 ID/PW로 카페24 로그인 폼 자동 제출
       → 카페24 로그인 세션 생성
```

**장점**: SSO 설정 불필요
**단점**: 임시 비밀번호 관리 필요, 프론트에서 폼 자동 제출은 불안정할 수 있음, 보안 우려

### 시나리오 C: 외부 세션 + 주문 시 연결

```
[사용자] → "구글로 가입" 버튼 클릭
       → Google OAuth 인증
       → 우리 서버에서 임시 회원 정보 저장 (우리 DB)
       → 쇼핑몰에서는 비로그인 상태로 이용
       → 주문 시점에 카페24 회원 생성 + 정보 입력
```

**장점**: 구현 가장 단순, 카페24 의존도 낮음
**단점**: 쇼핑몰에서 로그인 상태가 아님 → "회원가입"이라고 부르기 애매

## 4. 경쟁사 "1초가입" 추정 구현 방식

"1초가입"은 카페24 앱스토어 앱이고, 카카오 로그인으로 원클릭 가입을 지원합니다.
추정 플로우:

```
1. 앱 설치 시 ScriptTag API로 회원가입 페이지에 "카카오로 가입" 버튼 삽입
2. 사용자가 버튼 클릭 → 카카오 OAuth 인증
3. 카카오에서 받은 정보로 Admin API 회원 생성 (POST /admin/customers)
4. 생성된 회원으로 카페24 로그인 처리 (SSO 또는 자동 폼 제출)
5. 카카오 개인정보 동의 시 주소/연락처까지 자동 입력
```

이 앱이 요청하는 **카페24 API 권한**을 보면:
- 앱(읽기,쓰기) → ScriptTag 삽입
- 개인정보(읽기), 개인화정보(읽기) → 회원 정보 조회
- 프로모션(읽기,쓰기) → 쿠폰/적립금 발급
- 주문(읽기) → 주문 정보 확인

**"1초가입"의 카페24 API 권한 목록에 '회원(쓰기)'가 없다면**, Admin API로 직접 회원을 생성하는 것이 아니라 **카페24의 SNS/SSO 로그인 연동 기능을 활용**하는 것일 수 있습니다.

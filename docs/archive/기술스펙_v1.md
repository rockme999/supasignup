## 3. API 설계

### 3.1 OAuth Provider 엔드포인트 (카페24 SSO 용)

카페24가 호출하는 엔드포인트. 번개가입이 OAuth Provider 역할.

**카페24 SSO 등록 방식**: provider별 4개 SSO 항목 등록 (Google, Kakao, Naver, Apple).
각 항목의 Authorize URL: `bg.suparain.kr/oauth/authorize?provider=google` 등.
카페24가 플로우 시작점이 됨. Phase 2 추가 provider (Toss, Discord 등)는 카페24 SSO 5개 제한으로 ScriptTag 위젯 방식 사용.

**Account Linking**: 카페24 SSO 설정 시 "SNS 가입 시 동일 이메일 계정 연동" 옵션 활성화 안내 필수.

#### `GET /oauth/authorize`

카페24 SSO가 인증 요청 시 호출. provider가 지정되어 있으면 중간 페이지 없이 바로 소셜 OAuth로 리다이렉트.

**파라미터:**
| 파라미터 | 필수 | 설명 |
|---------|------|------|
| client_id | O | 쇼핑몰의 번개가입 Client ID |
| redirect_uri | O | 카페24 콜백 URL |
| response_type | O | "code" |
| state | O | CSRF 방지 |
| provider | O | "google" / "kakao" / "naver" / "apple" |

**처리 흐름:**
```typescript
// 1. client_id로 shop 조회
const shop = await getShopByClientId(clientId);

// 2. redirect_uri 화이트리스트 검증 (Open Redirect 방지)
const allowedUris = JSON.parse(shop.allowed_redirect_uris || '[]');
if (!allowedUris.includes(redirect_uri)) {
  return error(400, "Invalid redirect_uri");
}

// 3. provider가 해당 shop에서 활성화되어 있는지 확인
if (!shop.enabled_providers.includes(provider)) {
  return error(400, "Provider not enabled for this shop");
}

// 4. PKCE: code_verifier/code_challenge 생성
const codeVerifier = generateCodeVerifier();
const codeChallenge = await generateCodeChallenge(codeVerifier);

// 5. 세션 정보를 KV에 저장 (cafe24_state와 social_state 분리)
const socialState = crypto.randomUUID();
await KV.put(`oauth_session:${socialState}`, JSON.stringify({
  shop_id: shop.shop_id,
  redirect_uri,
  provider,
  cafe24_state: state,        // 카페24 SSO state (원본)
  social_state: socialState,  // 소셜 OAuth state (별도 생성)
}), { expirationTtl: 600 }); // 10분 TTL

// 6. PKCE code_verifier를 KV에 저장
await KV.put(`pkce:${socialState}`, codeVerifier, { expirationTtl: 600 });

// 7. 해당 소셜 서비스 OAuth URL로 즉시 리다이렉트 (PKCE 포함)
const oauthUrl = buildOAuthUrl(provider, {
  redirect_uri: `${BASE_URL}/oauth/callback/${provider}`,
  state: socialState,
  code_challenge: codeChallenge,
  code_challenge_method: 'S256',
});
return redirect(oauthUrl);
```

#### `GET /oauth/callback/:provider`

소셜 서비스 인증 완료 후 콜백. 사용자 정보를 저장하고 카페24로 돌아감.

**처리 흐름:**
```typescript
// 1. 소셜 서비스에서 authorization code 수신
// 2. KV에서 PKCE code_verifier 복원
const codeVerifier = await KV.get(`pkce:${state}`);
await KV.delete(`pkce:${state}`);

// 3. code → access_token 교환 (PKCE code_verifier 포함)
const socialToken = await exchangeCode(provider, code, codeVerifier);

// 4. access_token으로 사용자 정보 조회
const userInfo = await getUserInfo(provider, socialToken);

// 5. KV에서 세션 정보 복원
const session = await KV.get(`oauth_session:${state}`);

// 6. D1에 사용자 정보 저장/업데이트 (upsert)
//    ※ email, name, raw_data는 AES-GCM 암호화 후 저장
//    ※ email_hash (SHA-256)는 별도 인덱스로 저장 (검색용)
const user = await upsertUser(provider, userInfo);

// 7. shop_users 존재 여부로 가입/재로그인 구분
const existingShopUser = await getShopUser(session.shop_id, user.user_id);
const isNewSignup = !existingShopUser;

// 8. shop_users 매핑 생성 (신규 가입 시만)
if (isNewSignup) {
  await createShopUser(session.shop_id, user.user_id);
}

// 9. Authorization Code 생성 (카페24 SSO용)
const authCode = generateCode();
await KV.put(`auth_code:${authCode}`, JSON.stringify({
  user_id: user.user_id,
  shop_id: session.shop_id,
}), { expirationTtl: 600 });

// 10. 과금 체크 (매월 신규 가입 100명 한도, shop_users의 해당 월 count 기준)
if (isNewSignup) {
  const monthlyCount = await getMonthlySignupCount(session.shop_id);
  await checkBillingLimit(session.shop_id, monthlyCount);
}

// 11. 통계 기록
await recordStat(session.shop_id, user.user_id, provider, isNewSignup ? 'signup' : 'login');

// 12. 카페24 redirect_uri로 authorization code와 함께 리다이렉트
//     cafe24_state를 돌려줌 (소셜 state가 아닌 원본 카페24 state)
return redirect(`${session.redirect_uri}?code=${authCode}&state=${session.cafe24_state}`);
```

### 3.4 카페24 앱 설치 플로우

앱스토어에서 앱 설치 시 자동으로 쇼핑몰 등록 및 위젯 삽입을 처리.

**플로우:**
```
앱스토어 설치 클릭 → 카페24 OAuth 인증 → platform_access_token 발급
→ 번개가입 대시보드 자동 등록 → ScriptTag API로 위젯 JS 자동 삽입
```

#### `GET /api/cafe24/install`

카페24 앱스토어에서 앱 설치 시 호출되는 엔드포인트.

**처리 흐름:**
```typescript
// 1. 카페24에서 전달한 mall_id, shop 정보 수신
// 2. HMAC 서명 검증
// 3. 카페24 OAuth 인증 페이지로 리다이렉트
const authUrl = `https://${mallId}.cafe24api.com/api/v2/oauth/authorize?` +
  `response_type=code&client_id=${CAFE24_CLIENT_ID}&` +
  `redirect_uri=${BASE_URL}/api/cafe24/callback&` +
  `scope=mall.read_application,mall.write_application,mall.read_store,` +
  `mall.read_customer,mall.write_customer`;
return redirect(authUrl);
```

#### `GET /api/cafe24/callback`

카페24 OAuth 인증 완료 콜백. 쇼핑몰 자동 등록 및 위젯 삽입.

**처리 흐름:**
```typescript
// 1. authorization code → access_token/refresh_token 교환
const tokens = await cafe24Client.exchangeCode(code, mallId, redirectUri);

// 2. 쇼핑몰 정보 조회
const storeInfo = await cafe24Client.getStoreName(mallId, tokens.access_token);

// 3. D1에 쇼핑몰 자동 등록 (client_id/client_secret 생성)
const shop = await createShop({
  mall_id: mallId,
  platform: 'cafe24',
  shop_name: storeInfo.name,
  shop_url: `https://${mallId}.cafe24.com`,
  owner_id: ownerIdFromSession,
  platform_access_token: tokens.access_token,
  platform_refresh_token: tokens.refresh_token,
});

// 4. ScriptTag API로 위젯 JS 자동 삽입
await cafe24Client.createScriptTag(mallId, tokens.access_token,
  `${BASE_URL}/widget/buttons.js?shop=${shop.shop_id}`);

// 5. 대시보드로 리다이렉트 (설정 완료 페이지)
return redirect(`${BASE_URL}/dashboard/shops/${shop.shop_id}/setup`);
```

## 4. 소셜 OAuth 프로바이더 구현

### 4.1 공통 인터페이스

```typescript
interface OAuthProvider {
  name: string;                  // 'google' | 'kakao' | 'naver' | 'apple'
  displayName: string;           // '구글' | '카카오' | '네이버' | 'Apple'
  icon: string;                  // SVG 아이콘 또는 URL
  color: string;                 // 브랜드 컬러

  // OAuth URL 생성
  buildAuthUrl(params: {
    redirectUri: string;
    state: string;
    scope?: string;
  }): string;

  // Authorization Code → Access Token 교환
  exchangeCode(code: string, redirectUri: string): Promise<TokenResponse>;

  // Access Token으로 사용자 정보 조회
  getUserInfo(accessToken: string): Promise<UserInfo>;
}

interface UserInfo {
  provider: string;
  providerUid: string;     // 필수: 유니크 ID
  email?: string;           // 선택
  name?: string;            // 선택
  profileImage?: string;    // 선택
  rawData: Record<string, unknown>;  // 원본 데이터 전체
```

```typescript
// widget/src/buttons.ts

interface BGWidgetConfig {
  shopId: string;
  providers: string[];
  baseUrl: string;
}

class BGWidget {
  private config: BGWidgetConfig;
  private lastProvider: string | null;

  constructor() {
    // data-shop 속성에서 shopId 읽기
    this.loadConfig();
    this.lastProvider = localStorage.getItem('bg_last_provider');
  }

  async loadConfig() {
    // API 호출로 활성 프로바이더 목록 받기
    const res = await fetch(`${BASE_URL}/api/widget/config?shop=${this.shopId}`);
    this.config = await res.json();
    this.render();
  }

  render() {
    const container = document.getElementById('bg-signup-buttons')
      || this.findLoginPage();

    if (!container) return;

    // 프로바이더 정렬: lastProvider를 최상단으로
    const sorted = this.sortProviders(this.config.providers);

    // XSS 방어: innerHTML 대신 createElement + textContent 사용
    const widget = document.createElement('div');
    widget.className = 'bg-widget';

    const title = document.createElement('p');
    title.className = 'bg-title';
    title.textContent = '소셜 계정으로 간편 가입';
    widget.appendChild(title);

    sorted.forEach(p => widget.appendChild(this.renderButton(p)));

    const powered = document.createElement('p');
    powered.className = 'bg-powered';
    powered.textContent = 'powered by 번개가입 ⚡';
    widget.appendChild(powered);

    container.replaceChildren(widget);
    this.bindEvents();
  }

  renderButton(provider: string): HTMLButtonElement {
    const isLast = provider === this.lastProvider;
    const info = PROVIDER_INFO[provider];

    const btn = document.createElement('button');
    btn.className = isLast ? 'bg-btn bg-btn-highlight' : `bg-btn bg-btn-${provider}`;
    btn.dataset.provider = provider;

    const icon = document.createElement('span');
    icon.className = 'bg-icon';
    icon.textContent = info.icon;
    btn.appendChild(icon);

    if (isLast) {
      const flash = document.createElement('span');
      flash.className = 'bg-flash';
      flash.textContent = '⚡';
      btn.appendChild(flash);
    }

    const label = document.createElement('span');
    label.textContent = isLast ? `${info.displayName}로 로그인` : `${info.displayName}로 가입`;
    btn.appendChild(label);

    if (isLast) {
      const hint = document.createElement('span');
      hint.className = 'bg-hint';
      hint.textContent = '(지난번 사용)';
      btn.appendChild(hint);
    }

    return btn;
  }

  bindEvents() {
    document.querySelectorAll('.bg-btn').forEach(btn => {
      btn.addEventListener('click', () => {
        const provider = btn.dataset.provider;
        // OAuth 시작
        window.location.href =
          `${BASE_URL}/oauth/authorize?` +
          `provider=${provider}&` +
          `shop=${this.config.shopId}&` +
          `redirect_uri=${encodeURIComponent(window.location.href)}`;
      });
    });
  }

  // OAuth 완료 후 localStorage 업데이트
  static saveLastProvider() {
    const params = new URLSearchParams(window.location.search);
    const provider = params.get('bg_provider');
    if (provider) {
      localStorage.setItem('bg_last_provider', provider);
      // URL에서 파라미터 제거 (clean URL)
      params.delete('bg_provider');
      const cleanUrl = window.location.pathname +
        (params.toString() ? '?' + params.toString() : '');
      history.replaceState(null, '', cleanUrl);
    }
  }
}
```

## 7. 보안

### 7.1 인증/인가

| 대상 | 방식 |
|------|------|
| 카페24 SSO → 번개가입 | client_id + client_secret (Basic Auth) |
| 소셜 OAuth | 각 서비스의 OAuth 2.0 표준 |
| 대시보드 로그인 | 이메일 + 비밀번호 → bcrypt 해시 + JWT |
| 위젯 API | shop_id 기반 (공개, 민감 정보 없음) |

### 7.2 redirect_uri 검증 (H2)

- shops 테이블의 `allowed_redirect_uris` 필드에 허용된 URI 목록 저장 (JSON 배열)
- `/oauth/authorize` 에서 요청된 redirect_uri를 화이트리스트 대조 검증
- 일치하지 않으면 400 에러 반환 → Open Redirect 공격 방지

### 7.3 PKCE (H3)

- 소셜 OAuth 요청 시 code_verifier (랜덤 43-128자) 생성
- code_challenge = BASE64URL(SHA256(code_verifier))
- `/oauth/authorize`에서 code_challenge + code_challenge_method=S256을 소셜 OAuth URL에 포함
- `/oauth/callback`에서 token 교환 시 code_verifier를 포함하여 요청
- code_verifier는 KV에 `pkce:{state}` 키로 저장 (10분 TTL)

### 7.4 개인정보 암호화 (H4)

- users 테이블의 email, name, raw_data를 AES-GCM으로 암호화 저장
- 암호화 키: Workers Secret (`ENCRYPTION_KEY`)으로 관리
- 검색이 필요한 email은 `email_hash` (SHA-256) 인덱스를 별도 저장
- 플랫폼 토큰 (cafe24_access_token 등): AES-GCM 암호화 후 D1 저장
- 운영자 비밀번호: bcrypt 해시
- client_secret: 생성 시 1회만 표시, 이후 마스킹
- Authorization Code, Access Token: crypto.randomUUID() + KV TTL

### 7.5 Brute Force 방지 (M5)

- 대시보드 로그인 (`POST /api/dashboard/auth/login`)에 rate limiting 적용
- KV `login_attempt:{ip}` 키로 IP별 시도 횟수 추적 (5분 TTL)
- 5회 연속 실패 시 해당 IP 5분간 로그인 차단
- 성공 시 카운터 리셋

### 7.6 XSS 방어 (M6)

- 위젯에서 `innerHTML` 사용 금지
- `document.createElement()` + `textContent`로 DOM 조작
- 사용자 입력/외부 데이터는 반드시 textContent로 삽입

### 7.7 CSRF 방지

- OAuth state 파라미터: 모든 OAuth 플로우에 랜덤 state 필수
- 대시보드: SameSite=Strict 쿠키

### 7.8 state 파라미터 분리 (M7)

- 카페24 SSO state와 소셜 OAuth state를 별도 관리
- oauth_session KV에 `cafe24_state` (카페24 원본)와 `social_state` (소셜용 별도 생성) 분리 저장
- 카페24 콜백 시 `cafe24_state`를 반환, 소셜 콜백 시 `social_state`로 세션 조회

**서비스명**: 번개가입
**도메인**: bg.suparain.kr
**작성일**: 2026-03-12

---

## 1. 서비스 소개

### 1.1 서비스 정의
번개가입은 쇼핑몰에 소셜 로그인 기반 1-클릭 회원가입을 제공하는 B2B SaaS 서비스입니다.
쇼핑몰 운영자가 번개가입을 설치하면, 쇼핑몰 로그인 페이지에 소셜 가입 버튼이 추가되고, 방문자는 버튼 1번 클릭으로 회원가입과 로그인이 동시에 완료됩니다.

### 1.2 서비스 구성

```
번개가입 서비스
│
├── [위젯] 쇼핑몰에 삽입되는 소셜 가입 버튼 (사용자 대면)
│   ├── 스마트 버튼 (이전 사용 방법 기억/강조)
│   └── OAuth 인증 처리
│
├── [백엔드] OAuth Provider 서버 (카페24 SSO 연동)
│   ├── 인증 플로우 처리
│   ├── 사용자 정보 관리
│   └── 플랫폼 API 연동
│
└── [대시보드] 통합 관리자 페이지 (운영자 대면)
    ├── 쇼핑몰 관리 (멀티 플랫폼)
    ├── OAuth 프로바이더 선택/관리
    ├── 통계/분석
    └── 과금/플랜 관리
```

## 2. 사용자 시나리오

### 2.1 쇼핑몰 운영자 시나리오

#### 시나리오 A-0: 앱 설치 (카페24 앱스토어)

```
1. 운영자가 카페24 앱스토어에서 "번개가입" 검색/발견
2. "설치하기" 클릭
3. 카페24 OAuth 인증 화면 표시 (앱 권한 동의)
4. 인증 완료 → platform_access_token 발급
5. 번개가입 서버가 자동으로:
   - mall_id, 쇼핑몰 정보 저장
   - ScriptTag API 호출 → 위젯 JS 자동 등록
6. 번개가입 대시보드로 리다이렉트 (초기 설정 화면)
```

#### 시나리오 A: 서비스 설정 (최초 1회)

```
1. 앱 설치 후 대시보드 초기 설정 화면 진입
   (또는 bg.suparain.kr/dashboard 접속 → 이메일로 운영자 계정 생성)
2. 제공할 소셜 로그인 선택:
   ┌─────────────────────────────────────┐
   │  제공할 소셜 가입 방법을 선택하세요    │
   │                                     │
   │  ☑ Google    ☑ 카카오                │
   │  ☑ 네이버    ☐ Apple                 │
   │  ☐ 토스      ☐ Discord   (Phase 2)  │
   │  ☐ Telegram  ☐ TikTok   (Phase 2)  │
   │                                     │
   │  💡 최소 2개 이상 선택을 권장합니다    │
   └─────────────────────────────────────┘
3. SSO 설정 가이드 표시:
   - 선택한 provider별로 카페24 SSO에 각각 1개씩 등록 (총 4개까지)
   - 각 SSO 항목마다 입력할 정보:
     · Authorize URL: https://bg.suparain.kr/oauth/authorize?provider=google (등)
     · Token URL: https://bg.suparain.kr/oauth/token
     · UserInfo URL: https://bg.suparain.kr/oauth/userinfo
   - Client ID, Client Secret 자동 생성 (provider별 1쌍)
   - 복사 버튼 제공 + 단계별 스크린샷 가이드
   - Account Linking 설정 안내:
     · 카페24 관리자 > SSO 설정에서 "SNS 가입 시 동일 이메일 계정 연동" 옵션 활성화
     · 이를 통해 동일 이메일로 가입한 소셜 계정이 하나의 회원으로 연동됨
4. 운영자가 카페24 관리자에서 SSO 설정 완료 (provider별 SSO 항목 등록)
5. 번개가입 대시보드에서 "연동 확인" 클릭 → 설정 완료
```

#### 시나리오 B: 일상 운영

```
1. 대시보드 홈에서 오늘/이번 달 가입 현황 확인
2. 소셜별 가입 비율 확인 (구글 40%, 카카오 35%, 네이버 20%, 애플 5%)
3. 특정 소셜 서비스 추가/제거 가능
   예: TikTok이 추가되면 대시보드에서 활성화
4. 월 100명 한도 도달 시 → 유료 전환 안내
```

#### 시나리오 C: 멀티 쇼핑몰 운영 (Phase 3)

```
1. 운영자가 카페24 쇼핑몰 A + 아임웹 쇼핑몰 B를 운영
2. 대시보드에서 두 쇼핑몰 모두 등록/관리
3. 통합 통계에서 두 쇼핑몰 합산 데이터 확인
4. 쇼핑몰별 필터로 개별 통계 확인
5. 각 쇼핑몰마다 다른 소셜 프로바이더 설정 가능
   예: 쇼핑몰 A는 구글+카카오, 쇼핑몰 B는 카카오+네이버
```

#### 시나리오 H: 앱 해제/탈퇴

```
1. 운영자가 카페24 관리자에서 번개가입 앱 해제
2. 카페24가 앱 해제 Webhook 발송
3. 번개가입 서버 처리:
   - ScriptTag 자동 제거 (위젯 버튼 더 이상 표시 안 됨)
   - 쇼핑몰 상태를 "비활성" 으로 변경
   - 데이터 보관 정책: 30일간 보관 후 삭제
     (운영자가 기간 내 재설치 시 기존 설정 복원 가능)
4. 기존 소셜 가입 회원 처리:
   - 이미 가입된 회원의 카페24 계정은 유지됨 (카페24 자체 회원)
   - 소셜 로그인 버튼이 사라지므로 카페24 자체 로그인(이메일/비밀번호)으로 접속
   - 필요 시 카페24 "비밀번호 찾기"로 비밀번호 설정 안내
```

### 2.2 쇼핑몰 방문자 시나리오

#### 시나리오 D: 첫 방문 회원가입 (1-클릭)

```
1. 사용자가 쇼핑몰 로그인 페이지 방문
2. 운영자가 선택한 소셜 버튼들이 표시됨:
   ┌──────────────────────────────────────┐
   │  소셜 계정으로 간편 가입               │
   │                                      │
   │  [G] Google로 가입                    │
   │  [K] 카카오로 가입                     │
   │  [N] 네이버로 가입                     │
   │                                      │
   │  powered by 번개가입 ⚡               │
   └──────────────────────────────────────┘
   (운영자가 Apple을 비활성화했으므로 3개만 표시)

3. 사용자가 "카카오로 가입" 클릭
4. 카카오 인증 (이미 로그인 상태 → 자동 통과)
5. 카페24 약관 동의 팝업 (최초 1회)
6. 쇼핑몰에 로그인된 상태로 돌아옴
7. 브라우저 localStorage에 "kakao" 저장
```

#### 시나리오 E: 재방문 로그인 (스마트 버튼)

```
1. 사용자가 같은 쇼핑몰 재방문
2. 위젯이 localStorage에서 "kakao"를 읽음
3. 카카오 버튼이 강조 표시:
   ┌──────────────────────────────────────┐
   │  소셜 계정으로 간편 가입               │
   │                                      │
   │  ⚡ 카카오로 로그인 (지난번 사용)       │ ← 강조
   │  ─────────────────────────────────── │
   │  [G] Google로 가입                    │
   │  [N] 네이버로 가입                     │
   │                                      │
   │  powered by 번개가입 ⚡               │
   └──────────────────────────────────────┘

4. 사용자가 강조된 버튼 클릭 → 즉시 로그인
```

#### 시나리오 F: 다른 쇼핑몰 방문 (크로스 쇼핑몰)

```
1. 사용자가 A쇼핑몰에서 카카오로 가입 (localStorage: "kakao")
2. B쇼핑몰(번개가입 사용) 방문 시에도 카카오 버튼 강조
3. 클릭 한 번으로 B쇼핑몰에도 가입
```

#### 시나리오 G: 주문 시 추가 정보 입력 (Progressive Profiling)

```
1. 소셜 가입으로 로그인 된 사용자가 상품 주문
2. 주문 페이지에서 배송지, 전화번호 등 추가 입력
3. 입력된 정보가 카페24 회원 정보에 자동 업데이트
4. 다음 주문부터는 저장된 정보 자동 채움
```

## 3. 기능 상세

### 3.1 위젯 (쇼핑몰 삽입 버튼)

#### 3.1.1 버튼 표시 로직

```
위젯 로딩 시:
1. bg.suparain.kr에서 해당 쇼핑몰의 활성 프로바이더 목록 조회
2. localStorage에서 bg_last_provider 확인
3. 활성 프로바이더 목록에 포함된 것만 버튼 렌더링
4. bg_last_provider가 있으면 해당 버튼을 최상단 + 강조
5. 나머지 버튼은 기본 스타일로 아래에 표시
```

#### 3.1.2 버튼 디자인

**기본 스타일 (운영자가 변경 불가 - MVP):**
- 각 소셜 서비스의 공식 브랜드 컬러/아이콘 사용
- Google: 흰 배경 + G 로고
- 카카오: 노란 배경 + 카카오 로고
- 네이버: 초록 배경 + N 로고
- Apple: 검은 배경 + Apple 로고

**강조 스타일 (스마트 버튼):**
- 기본 스타일 + 번개 아이콘(⚡) + 파란 테두리 + "지난번 사용" 라벨
- 상단 배치 + 크기 약간 확대

#### 3.1.3 위젯 삽입 방식

**카페24 (MVP):** ScriptTag API로 자동 삽입
- 앱 설치 시 ScriptTag API 호출로 모든 페이지에 JS 삽입
- JS가 로그인/회원가입 페이지를 감지하여 버튼 렌더링

**기타 플랫폼 (Phase 3):** 코드 스니펫 수동 삽입
```html
<div id="bg-signup-buttons"></div>
<script src="https://bg.suparain.kr/widget/buttons.js"
        data-shop="SHOP_ID"></script>
```

### 3.4 과금 시스템

#### 3.4.1 사용량 체크 로직

```
회원가입 요청 시:
1. 해당 쇼핑몰의 당월 신규 가입 수 확인 (매월 1일 초기화)
2. shop owner의 subscription 확인
3. 무료 플랜 + 해당 월 신규 가입 수 >= 100 → 번개가입 버튼 비활성화/숨김 + 운영자에게 알림
   - 기존 회원 로그인은 카페24 자체 로그인(이메일/비밀번호)으로 가능 (영향 없음)
4. 유료 플랜 → 무제한 처리
5. 가입 성공 시 당월 신규 가입 수 += 1
```

#### 3.4.2 유료 전환 유도

- 당월 신규 가입 80명 도달 시: 대시보드에 안내 배너
- 당월 신규 가입 95명 도달 시: 이메일 알림
- 당월 신규 가입 100명 도달 시: 번개가입 버튼 비활성화/숨김 + 대시보드/이메일 알림 + 유료 전환 버튼

#### 3.4.3 과금/결제 방법

```
결제 방식: 카페24 수동 인앱결제 (모든 플랫폼 동일)

플로우:
1. 운영자가 대시보드에서 "유료 플랜 전환" 클릭
2. 카페24 인앱결제 API를 통해 결제 요청 생성
3. 운영자가 카페24 결제 화면에서 결제 완료
4. 결제 완료 Webhook 수신 → 플랜 업그레이드 처리
5. 월간/연간 자동 갱신 (카페24 인앱결제 정기결제)

가격:
- 무료 플랜: 쇼핑몰당 매월 신규 가입 100명
- 유료 플랜 (월간): 29,900원/월 - 무제한
- 유료 플랜 (연간): 329,900원/년 - 무제한 (약 8% 할인)
```
